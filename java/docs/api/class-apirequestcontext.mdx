此 API 用於 Web API 測試。您可以使用它來觸發 API 端點、組態設定微服務、準備環境或服務以進行端對端測試。

每個 Playwright 瀏覽器情境都有一個關聯的 [APIRequestContext] 實例，它與瀏覽器情境共享 cookie 儲存，並可透過 [BrowserContext.request()](/api/class-browsercontext.mdx#browser-context-request) 或 [Page.request()](/api/class-page.mdx#page-request) 存取。也可以透過呼叫 [APIRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context) 手動建立新的 `APIRequestContext` 實例。

**Cookie 管理**

由 [BrowserContext.request()](/api/class-browsercontext.mdx#browser-context-request) 和 [Page.request()](/api/class-page.mdx#page-request) 傳回的 [APIRequestContext] 與對應的 [BrowserContext] 共享 cookie 儲存。每個 API 請求都會有 `Cookie` 標頭，填入來自瀏覽器情境的值。如果 API 回應包含 `Set-Cookie` 標頭，它會自動更新 [BrowserContext] cookies，並且從頁面發出的請求會取得它們。這意味著如果您使用此 API 登入，您的端對端測試將會登入，反之亦然。

如果您想要 API 請求不干擾瀏覽器 cookies，您應該透過呼叫 [APIRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context) 建立新的 [APIRequestContext]。這樣的 `APIRequestContext` 物件將擁有自己的隔離 cookie 儲存。

### fetch {#api-request-context-fetch}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.fetch</x-search>

傳送 HTTP(S) 請求並傳回其回應。此方法會從情境填入請求 cookies，並從回應更新情境 cookies。此方法會自動跟隨重新導向。

**使用方式**

JSON 物件可以直接傳遞給請求：

```java
Map<String, Object> data = new HashMap();
data.put("title", "Book Title");
data.put("body", "John Doe");
request.fetch("https://example.com/api/createBook", RequestOptions.create().setMethod("post").setData(data));
```

在請求主體中傳送檔案的常見方式是透過指定 `multipart` 參數，將它們作為 `multipart/form-data` 編碼的表單欄位上傳：

```java
// Pass file path to the form data constructor:
Path file = Paths.get("team.csv");
APIResponse response = request.fetch("https://example.com/api/uploadTeamList",
  RequestOptions.create().setMethod("post").setMultipart(
    FormData.create().set("fileField", file)));

// 或者您可以直接將檔案內容作為 FilePayload 物件傳遞：
FilePayload filePayload = new FilePayload("f.js", "text/javascript",
      "console.log(2022);".getBytes(StandardCharsets.UTF_8));
APIResponse response = request.fetch("https://example.com/api/uploadScript",
  RequestOptions.create().setMethod("post").setMultipart(
    FormData.create().set("fileField", filePayload)));
```

**參數**
- `urlOrRequest` [String] | [Request]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-fetch-option-url-or-request"/><a href="#api-request-context-fetch-option-url-or-request" class="list-anchor">#</a>
  
  目標 URL 或要從中取得所有參數的 Request。
- `options` [RequestOptions] *(選用)* <font size="2">Added in: v1.18</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-fetch-option-params"/><a href="#api-request-context-fetch-option-params" class="list-anchor">#</a>
  
  Optional request parameters.

**傳回值**

### post {#api-request-context-post}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.post</x-search>

傳送 HTTP(S) [POST](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/POST) 請求並傳回其回應。此方法會從情境填入請求 cookies，並從回應更新情境 cookies。此方法會自動跟隨重新導向。

**使用方式**

JSON 物件可以直接傳遞給請求：

```java
Map<String, Object> data = new HashMap();
data.put("title", "Book Title");
data.put("body", "John Doe");
request.post("https://example.com/api/createBook", RequestOptions.create().setData(data));
```

要將表單資料傳送到伺服器，請使用 `form` 選項。其值會使用 `application/x-www-form-urlencoded` 編碼編碼到請求主體中（請參閱下方如何使用 `multipart/form-data` 表單編碼傳送檔案）：

```java
request.post("https://example.com/api/findBook", RequestOptions.create().setForm(
    FormData.create().set("title", "Book Title").set("body", "John Doe")
));
```

在請求主體中傳送檔案的常見方式是將它們作為具有 `multipart/form-data` 編碼的表單欄位上傳。使用 [FormData] 建構請求主體，並將其作為 `multipart` 參數傳遞給請求：

```java
// Pass file path to the form data constructor:
Path file = Paths.get("team.csv");
APIResponse response = request.post("https://example.com/api/uploadTeamList",
  RequestOptions.create().setMultipart(
    FormData.create().set("fileField", file)));

// 或者您可以直接將檔案內容作為 FilePayload 物件傳遞：
FilePayload filePayload1 = new FilePayload("f1.js", "text/javascript",
      "console.log(2022);".getBytes(StandardCharsets.UTF_8));
APIResponse response = request.post("https://example.com/api/uploadScript",
  RequestOptions.create().setMultipart(
    FormData.create().set("fileField", filePayload)));
```

**參數**
- `url` [String]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-post-option-url"/><a href="#api-request-context-post-option-url" class="list-anchor">#</a>
  
  目標 URL。
- `options` [RequestOptions] *(選用)* <font size="2">Added in: v1.18</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-post-option-params"/><a href="#api-request-context-post-option-params" class="list-anchor">#</a>

APIRequestContext.storageState();
