## 簡介

Playwright Java 不是執行緒安全的，也就是說它的所有方法，以及由它創建的所有物件（如 [`BrowserContext`]、[`Browser`]、[`Page`] 等）上的方法，都應該在創建 Playwright 物件的同一個執行緒上呼叫，或者應該實作適當的同步化機制，以確保在任何給定時間內只有一個執行緒呼叫 Playwright 方法。話雖如此，在各自的執行緒上創建多個 Playwright 執行個體是可以的。

以下是一個範例，其中三個 playwright 執行個體分別在各自的執行緒上創建。每個執行個體啟動自己的瀏覽器程序並對其執行測試。

```java
package org.example;

import com.microsoft.playwright.*;

import java.nio.file.Paths;

import static java.util.Arrays.asList;

public class PlaywrightThread extends Thread {
  private final String browserName;

  private PlaywrightThread(String browserName) {
    this.browserName = browserName;
  }

  public static void main(String[] args) throws InterruptedException {
    // 為每個瀏覽器創建單獨的 playwright 執行緒。
    for (String browserName: asList("chromium", "webkit", "firefox")) {
      Thread thread = new PlaywrightThread(browserName);
      thread.start();
    }
  }

  @Override
  public void run() {
    try (Playwright playwright = Playwright.create()) {
      BrowserType browserType = getBrowserType(playwright, browserName);
      Browser browser = browserType.launch();
      Page page = browser.newPage();
      page.navigate("https://playwright.dev/");
      page.screenshot(new Page.ScreenshotOptions().setPath(Paths.get("user-agent-" + browserName + ".png")));
    }
  }

  private static BrowserType getBrowserType(Playwright playwright, String browserName) {
    switch (browserName) {
      case "chromium":
        return playwright.chromium();
      case "webkit":
        return playwright.webkit();
      case "firefox":
        return playwright.firefox();
      default:
        throw new IllegalArgumentException();
    }
  }
}
```

## 同步 API 和事件分派

在同步的 Playwright API 中，所有事件只有在 Playwright 執行其訊息迴圈時才會被分派。這會在您呼叫任何 API 方法時自動發生，如果堆疊上沒有作用中的 Playwright 呼叫則不會發生。如果您需要等待事件，最好的方法是透過其中一個 `waitFor*` 方法。

### `Page.waitForTimeout()` vs. `Thread.sleep()`

同步 API 的一個結果是，如果您因為任何原因呼叫 `Thread.sleep()`，在執行緒休眠期間不會觸發任何事件。如果您希望在程式執行暫停期間分派來自瀏覽器的事件，請使用 [`Page.waitForTimeout()`](/api/class-page.mdx#page-wait-for-timeout) 或 [`Frame.waitForTimeout()`](/api/class-frame.mdx#frame-wait-for-timeout)：

```java
page.onResponse(response -> System.out.println(response.url()));
page.navigate("https://playwright.dev");
System.out.println("-- did navigate --");
// 阻塞當前執行緒 60 秒並確保事件被分派。
page.waitForTimeout(60_000);
```
