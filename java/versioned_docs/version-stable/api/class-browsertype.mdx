`BrowserType` 提供方法來啟動特定的瀏覽器實例或連接到現有實例。以下是使用 Playwright 進行自動化的典型範例：

```java
import com.microsoft.playwright.*;

public class Example {
  public static void main(String[] args) {
    try (Playwright playwright = Playwright.create()) {
      BrowserType chromium = playwright.chromium();
      瀏覽器 瀏覽器 = chromium.啟動();
      Page page = 瀏覽器.newPage();
      page.navigate("https://example.com");
      // other actions...
      瀏覽器.close();
    }
  }
}
```

## 方法

### connect {#browser-type-connect}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>browserType.connect</x-search>

此方法將 Playwright 連接到透過 Node.js 中的 `BrowserType.launchServer` 建立的現有瀏覽器實例。

:::note

連接的 Playwright 實例的主要和次要版本需要與啟動瀏覽器的 Playwright 版本相符（1.2.3 → 與 1.2.x 相容）。
:::

**使用方式**

```java
BrowserType.連接(wsEndpoint);
BrowserType.連接(wsEndpoint, options);
```

**參數**
- `wsEndpoint` [String] <font size="2">Added in: v1.10</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-選項-ws-endpoint"/><a href="#browser-type-connect-option-ws-endpoint" class="list-anchor">#</a>
  
  A Playwright 瀏覽器 WebSocket 端點 to 連接 to. You obtain this 端點 via `BrowserServer.wsEndpoint`.
- `options` `BrowserType.ConnectOptions` *(選擇性)*
  - `setExposeNetwork` [String] *(optional)* <font size="2">Added in: v1.37</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-選項-expose-網路"/><a href="#browser-type-connect-option-expose-網路" class="list-anchor">#</a>
    
    此選項exposes 網路 available on the connecting client to 瀏覽器being connected to. Consists of a list of rules separated by comma.
    
    Available rules:
    1. Hostname pattern, 例如: `example.com`, `*.org:99`, `x.*.y.com`, `*foo.org`.
    1. IP literal, 例如: `127.0.0.1`, `0.0.0.0:99`, `[::1]`, `[0:0::1]:99`.
    1. `<loopback>` 比對本地回送介面：`localhost`、`*.localhost`、`127.0.0.1`、`[::1]`。
    
    Some common examples:
    1. `"*"` to expose all 網路.
    1. `"<loopback>"` to expose localhost 網路.
    1. `"*.test.internal-domain,*.staging.internal-domain,<loopback>"` 用於暴露測試/預備部署和 localhost。
  - `setHeaders` [Map]&lt;[String], [String]&gt; *(optional)* <font size="2">Added in: v1.11</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-選項-headers"/><a href="#browser-type-connect-option-headers" class="list-anchor">#</a>
    
    Additional HTTP headers to be sent with web socket 連接 request. Optional.
  - `setSlowMo` [double] *(optional)* <font size="2">Added in: v1.10</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-選項-slow-mo"/><a href="#browser-type-connect-option-slow-mo" class="list-anchor">#</a>
    
    Slows down Playwright operations by the specified amount of milliseconds. Useful so that you can see what is going on. 預設為0.
  - `setTimeout` [double] *(optional)* <font size="2">Added in: v1.10</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-選項-timeout"/><a href="#browser-type-connect-option-timeout" class="list-anchor">#</a>
    
    Maximum time in milliseconds to wait for 連接to be established. 預設為`0` (no 逾時).

**回傳值**
- [Browser]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-return"/><a href="#瀏覽器-type-連接-return" class="list-anchor">#</a>

### connectOverCDP {#browser-type-connect-over-cdp}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font><x-search>browserType.connectOverCDP</x-search>

此方法使用 Chrome DevTools Protocol 將 Playwright 連接到現有的瀏覽器實例。

預設瀏覽器情境可透過 [Browser.contexts()](/api/class-browser.mdx#browser-contexts) 存取。

:::note

透過 Chrome DevTools Protocol 連接僅支援基於 Chromium 的瀏覽器。
:::

:::note
此連接的保真度遠低於透過 [BrowserType.connect()](/api/class-browsertype.mdx#browser-type-connect) 的 Playwright 協定連接。如果您遇到問題或嘗試使用進階功能，您可能會想要使用 [BrowserType.connect()](/api/class-browsertype.mdx#browser-type-connect)。
:::

**使用方式**

```java
瀏覽器 瀏覽器 = playwright.chromium().connectOverCDP("http://localhost:9222");
BrowserContext defaultContext = 瀏覽器.contexts().get(0);
Page page = defaultContext.pages().get(0);
```

**參數**
- `endpointURL` [String] <font size="2">Added in: v1.11</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-over-cdp-選項-endpoint-url"/><a href="#browser-type-connect-over-cdp-option-endpoint-url" class="list-anchor">#</a>
  
  A CDP WebSocket 端點 or http url to 連接 to. 例如 `http://localhost:9222/` or `ws://127.0.0.1:9222/devtools/browser/387adf4c-243f-4051-a181-46798f4a46f4`.
- `options` `BrowserType.ConnectOverCDPOptions` *(選擇性)*
  - `setHeaders` [Map]&lt;[String], [String]&gt; *(optional)* <font size="2">Added in: v1.11</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-over-cdp-選項-headers"/><a href="#browser-type-connect-over-cdp-option-headers" class="list-anchor">#</a>
    
    Additional HTTP headers to be sent with 連接 request. Optional.
  - `setSlowMo` [double] *(optional)* <font size="2">Added in: v1.11</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-over-cdp-選項-slow-mo"/><a href="#browser-type-connect-over-cdp-option-slow-mo" class="list-anchor">#</a>
    
    Slows down Playwright operations by the specified amount of milliseconds. Useful so that you can see what is going on. 預設為0.
  - `setTimeout` [double] *(optional)* <font size="2">Added in: v1.11</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-over-cdp-選項-timeout"/><a href="#browser-type-connect-over-cdp-option-timeout" class="list-anchor">#</a>
    
    Maximum time in milliseconds to wait for 連接to be established. 預設為`30000` (30 seconds). Pass `0` to disable 逾時.

**回傳值**
- [Browser]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-連接-over-cdp-return"/><a href="#瀏覽器-type-連接-over-cdp-return" class="list-anchor">#</a>

### launch {#browser-type-launch}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>browserType.啟動</x-search>

回傳瀏覽器實例。

**使用方式**

You can use [setIgnoreDefaultArgs](/api/class-browsertype.mdx#browser-type-launch-option-ignore-default-args) to filter out `--mute-audio` from 預設 arguments:

```java
// Or "firefox" or "webkit".
瀏覽器 瀏覽器 = chromium.啟動(new BrowserType.LaunchOptions()
  .setIgnoreDefaultArgs(Arrays.asList("--mute-audio")));
```

> **Chromium-only** Playwright can also be used to control the Google Chrome or Microsoft Edge browsers, but it works best with the 版本 of Chromium it is bundled with. There is no guarantee it will work with any other 版本. Use [setExecutablePath](/api/class-browsertype.mdx#browser-type-launch-option-executable-path) 選項 with extreme caution.
>
> 如果偏好使用 Google Chrome（而非 Chromium），建議使用 [Chrome Canary](https://www.google.com/chrome/browser/canary.html) 或 [Dev Channel](https://www.chromium.org/getting-involved/dev-channel) 建置版本。
>
> Stock browsers like Google Chrome and Microsoft Edge are suitable for tests that require proprietary media codecs for 影片 playback. See [this article](https://www.howtogeek.com/202825/what%E2%80%99s-the-difference-between-chromium-and-chrome/) for other differences between Chromium and Chrome. [This article](https://chromium.googlesource.com/chromium/src/+/lkgr/docs/chromium_browser_vs_google_chrome.md) describes some differences for Linux users.

### launchPersistentContext {#瀏覽器-type-啟動-持久-情境}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>browserType.launchPersistentContext</x-search>

回傳持久瀏覽器情境實例。

啟動使用位於 [userDataDir](/api/class-browsertype.mdx#browser-type-launch-persistent-context-option-user-data-dir) 的持久儲存的瀏覽器，並回傳唯一的情境。關閉此情境會自動關閉瀏覽器。

**使用方式**

```java
BrowserType.launchPersistentContext(userDataDir);
BrowserType.launchPersistentContext(userDataDir, options);
```

**參數**
- `userDataDir` [Path]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-launch-persistent-context-option-user-data-dir"/><a href="#browser-type-launch-persistent-context-option-user-data-dir" class="list-anchor">#</a>
  
  到...的路徑a User Data 目錄, which stores 瀏覽器 工作階段 data like cookies and local 儲存. Pass an empty string to create a temporary 目錄.
  
  更多詳細資訊for [Chromium](https://chromium.googlesource.com/chromium/src/+/master/docs/user_data_dir.md#introduction) and [Firefox](https://wiki.mozilla.org/Firefox/CommandLineOptions#User_profile). Chromium's user data 目錄 is the **parent** 目錄 of the "設定檔 路徑" seen at `chrome://version`.
  
  請注意browsers do not allow launching multiple instances with the same User Data 目錄.
  
  :::warning
  
  Chromium/Chrome: Due to recent Chrome policy changes, automating the 預設 Chrome user 設定檔 is not supported. Pointing `userDataDir` to Chrome's main "User Data" 目錄 (the 設定檔 used for your regular browsing) may result in pages not loading or 瀏覽器exiting. Create and use a separate 目錄 (例如, an empty 資料夾) as your automation 設定檔 instead. See https://developer.chrome.com/blog/remote-debugging-port for details.
  :::
  
- `options` `BrowserType.LaunchPersistentContextOptions` *(選擇性)*
  - `setAcceptDownloads` [boolean] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-accept-downloads"/><a href="#browser-type-launch-persistent-context-option-accept-downloads" class="list-anchor">#</a>
    
    是否automatically 下載 all the attachments. 預設為`true` where all the downloads are accepted.
  - `setArgs` [List]&lt;[String]&gt; *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-args"/><a href="#browser-type-launch-persistent-context-option-args" class="list-anchor">#</a>
    
    :::warning
    
    Use custom 瀏覽器 args at your own risk, as some of them may break Playwright functionality.
    :::
    
    Additional arguments to pass to 瀏覽器instance. The list of Chromium flags can be found [here](https://peter.sh/experiments/chromium-command-line-switches/).
  - `setBaseURL` [String] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-base-url"/><a href="#browser-type-launch-persistent-context-option-base-url" class="list-anchor">#</a>
    
    When using [Page.navigate()](/api/class-page.mdx#page-goto), [Page.route()](/api/class-page.mdx#page-route), [Page.waitForURL()](/api/class-page.mdx#page-wait-for-url), [Page.waitForRequest()](/api/class-page.mdx#page-wait-for-request), or [Page.waitForResponse()](/api/class-page.mdx#page-wait-for-response) it takes the base URL in consideration by using the [__CODE_BLOCK_0__](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) constructor for building the corresponding URL. Unset 預設情況下. Examples:
    * baseURL：`http://localhost:3000` 並導航到 `/bar.html` 會得到 `http://localhost:3000/bar.html`
    * baseURL：`http://localhost:3000/foo/` 並導航到 `./bar.html` 會得到 `http://localhost:3000/foo/bar.html`
    * baseURL：`http://localhost:3000/foo`（無尾隨斜線）並導航到 `./bar.html` 會得到 `http://localhost:3000/bar.html`

  - `setSlowMo` [double] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-slow-mo"/><a href="#browser-type-launch-persistent-context-option-slow-mo" class="list-anchor">#</a>
    
    將 Playwright 操作減慢指定的毫秒數。這對於觀察正在進行的操作很有用。
  - `setStrictSelectors` [boolean] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-strict-selectors"/><a href="#browser-type-launch-persistent-context-option-strict-selectors" class="list-anchor">#</a>
    
    若設定為 `true`，會為此情境啟用嚴格選擇器模式. In the strict selectors mode all operations on selectors that imply single target DOM element will throw when more than one element matches the selector. 此選項does not affect any Locator APIs (Locators are always strict). 預設為`false`. See [Locator] to 深入了解about the strict mode.
  - `setTimeout` [double] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-timeout"/><a href="#browser-type-launch-persistent-context-option-timeout" class="list-anchor">#</a>
    
    等待瀏覽器實例啟動的最長時間（以毫秒為單位）. 預設為`30000` (30 seconds). Pass `0` to disable 逾時.
  - `setTimezoneId` [String] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-timezone-id"/><a href="#browser-type-launch-persistent-context-option-timezone-id" class="list-anchor">#</a>
    
    變更情境的時區. See [ICU's metaZones.txt](https://cs.chromium.org/chromium/src/third_party/icu/source/data/misc/metaZones.txt?rcl=faee8bc70570192d82d2978a71e2a615788597d1) for a list of supported 時區 IDs. 預設為the system 時區.
  - `setTracesDir` [路徑] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-traces-dir"/><a href="#browser-type-launch-persistent-context-option-traces-dir" class="list-anchor">#</a>
    
    若指定，追蹤檔案將儲存到此目錄.
  - `setUserAgent` [String] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-user-agent"/><a href="#browser-type-launch-persistent-context-option-user-agent" class="list-anchor">#</a>
    
    此情境要使用的特定 user agent.
  - `setViewportSize` [null] | ViewportSize *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="browser-type-啟動-持久-情境-選項-檢視區"/><a href="#browser-type-launch-persistent-context-option-檢視區" class="list-anchor">#</a>
    - `setWidth` [int]
      
      頁面寬度（以像素為單位）。
    - `setHeight` [int]
      
      頁面高度（以像素為單位）。
    
    為每個頁面模擬一致的檢視區. 預設為 1280x720 檢視區. Use `null` to disable the consistent 檢視區 模擬. 深入了解about [viewport emulation](../emulation#viewport).
    
    :::note
    
    The `null` value opts out from the 預設 presets, 使檢視區取決於主機視窗大小 defined by the operating system. It makes the execution of the tests non-deterministic.
    :::
