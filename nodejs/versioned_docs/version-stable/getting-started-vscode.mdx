## 簡介

Playwright VS Code 擴充功能將 Playwright Test 的強大功能直接帶入您的編輯器，讓您能夠透過無縫的 UI 驅動體驗來執行、偵錯和產生測試。本指南將帶您了解如何設定擴充功能，以及使用其核心功能來增強您的端對端測試工作流程。

<LiteYouTube id="WvsLGZnHmzw" title="Getting Started with Playwright in VS Code" />

## 先決條件

在開始之前，請確保您已安裝以下軟體：
- [Node.js](https://nodejs.org/)（建議使用 LTS 版本）
- [Visual Studio Code](https://code.visualstudio.com/)

## 快速入門

### 安裝與設定
1.  **安裝擴充功能**：在 VS Code 中開啟擴充功能檢視（`Ctrl+Shift+X` 或 `Cmd+Shift+X`），搜尋「Playwright」。[安裝來自 Microsoft 的官方擴充功能](https://marketplace.visualstudio.com/items?itemName=ms-playwright.playwright)。

<ProgressiveImage image={require("../images/getting-started/vscode-extension.png")} alt="install playwright extension" />
1.  **安裝 Playwright**：安裝擴充功能後，開啟命令選擇區（`Ctrl+Shift+P` 或 `Cmd+Shift+P`）並執行 **Test: Install Playwright** 命令。

<ProgressiveImage image={require("../images/getting-started/install-playwright.png")} alt="install playwright" />

3.  **選擇瀏覽器**：選擇您測試時想要使用的瀏覽器（例如 Chromium、Firefox、WebKit）。您也可以新增 GitHub Actions 工作流程來在 CI 中執行測試。這些設定稍後可以在您的 `playwright.config.ts` 檔案中變更。

<ProgressiveImage image={require("../images/getting-started/install-browsers.png")} alt="install browsers" />

### 開啟測試側邊欄

點選 VS Code 活動列中的**測試圖示**來開啟測試檔案總管。在這裡，您可以找到您的測試，以及用於管理專案、工具和設定的 Playwright 側邊欄。

<ProgressiveImage image={require("../images/getting-started/testing-sidebar.png")} alt="Testing Sidebar" />

## 核心功能

### 執行您的測試

<LiteYouTube id="mQmcIBMsc38" title="Running Playwright Tests in VS Code" />
-   **執行單一測試**：點選任何測試旁邊的綠色「播放」圖示來執行它。如果測試通過，播放按鈕會變成綠色勾選標記，如果測試失敗則會變成紅色 X。您可以看到測試名稱旁邊顯示的測試執行時間。此外，測試結果面板會自動在 VS Code 底部開啟，顯示測試執行摘要，包括執行了多少測試、有多少通過、失敗或被跳過，以及總執行時間。

<ProgressiveImage image={require("../images/getting-started/run-single-test.png")} alt="run a single test" />
-   **執行所有測試**：您可以在不同層級執行所有測試。點選特定測試檔案旁邊的播放圖示來執行該檔案內的所有測試，或點選測試檔案總管最頂端的播放圖示來執行整個專案中的所有測試。

<ProgressiveImage image={require("../images/getting-started/run-all-tests.png")} alt="run all tests" />
-   **在多個瀏覽器上執行**：在 Playwright 側邊欄中，勾選您想要測試的專案（瀏覽器）的核取方塊。Playwright 中的專案代表不同的瀏覽器設定 - 每個專案通常對應到一個特定的瀏覽器（如 Chromium、Firefox 或 WebKit），並具有自己的設定，例如檢視區大小、裝置模擬或其他特定於瀏覽器的選項。當您執行測試時，它會在所有選定的專案中執行，讓您可以驗證您的應用程式在不同瀏覽器和設定中的一致性。

<ProgressiveImage image={require("../images/getting-started/select-projects.png")} alt="Selecting projects to run tests on" />
-   **顯示瀏覽器**：要在即時瀏覽器視窗中觀看您的測試執行，請在側邊欄中啟用**顯示瀏覽器**選項。停用它則會在無頭模式下執行（測試在背景執行，不會開啟可見的瀏覽器視窗）。

<ProgressiveImage image={require("../images/getting-started/show-browser.png")} alt="show browsers while running tests" />

### 偵錯您的測試

<LiteYouTube id="tJF7UhA59Gc" title="Debugging Playwright tests in VS Code" />

VS Code 擴充功能提供強大的偵錯工具來幫助您識別和修復測試中的問題。您可以設定中斷點、檢查變數、檢視詳細的錯誤訊息、取得 AI 驅動的建議來解決測試失敗，以及使用全面的追蹤檢視器來逐步分析測試執行。
-   **使用中斷點**：點選行號旁邊的空白處來設定中斷點。右鍵點選測試並選擇**偵錯測試**。測試會在您的中斷點處暫停，讓您可以檢查變數並逐步執行程式碼。
  
  <ProgressiveImage image={require("../images/getting-started/debug-mode.png")} alt="setting debug mode" />
-   **即時偵錯**：啟用**顯示瀏覽器**後，點選您程式碼中的定位器。Playwright 會在瀏覽器中反白顯示對應的元素，讓您輕鬆驗證定位器。
  
  <ProgressiveImage image={require("../images/getting-started/live-debugging.png")} alt="live debugging in vs code" />
-   **檢視錯誤訊息**：如果測試失敗，擴充功能會直接在編輯器中顯示詳細的錯誤訊息，包括預期值與實際值的比較，以及完整的呼叫記錄。

<ProgressiveImage image={require("../images/getting-started/error-messaging.png")} alt="error messaging in vs code" />
-   **使用 AI 修復**：當測試失敗時，點選錯誤旁邊的星星圖示，即可從 Copilot 取得 AI 驅動的修復建議。Copilot 會分析錯誤並建議程式碼變更來解決問題。

<ProgressiveImage image={require("../images/getting-started/fix-with-ai.png")} alt="fix with ai in vs code" />
-   **使用追蹤檢視器進行偵錯**：若要進行全面偵錯，請在 Playwright 側邊欄中啟用**顯示追蹤檢視器**選項。當您的測試完成時，詳細的追蹤會自動開啟，為您提供測試執行的完整時間軸。追蹤檢視器特別適用於：
  - **逐步分析**：以精確的時間戳記導覽您的測試執行的每個動作
  - **DOM 檢查**：檢視測試執行期間任何時點的 DOM 快照，查看頁面當時的確切狀態
  - **網路監控**：檢查測試期間發生的所有網路請求和回應
  - **主控台記錄**：存取來自瀏覽器的所有主控台訊息和錯誤
  - **來源對應**：直接跳轉到執行每個動作的來源程式碼
  - **視覺偵錯**：查看螢幕截圖並了解使用者在每個步驟中看到的內容
  
  追蹤檢視器在偵錯不穩定的測試或了解複雜的使用者互動時特別有價值。

<ProgressiveImage image={require("../images/getting-started/trace-viewer-debug.png")} alt="trace viewer debugging" />

若要了解更多資訊，請參閱我們的[追蹤檢視器指南](./trace-viewer.mdx)。

### 使用 CodeGen 產生測試

CodeGen 是 Playwright 強大的測試產生工具，它會透過錄製您與網頁的互動來自動建立測試程式碼。您無需從頭撰寫測試，只需在 CodeGen 擷取您的動作並將其轉換為具有適當定位器和斷言的可靠測試程式碼時，瀏覽您的應用程式即可。

<LiteYouTube id="5XIZPqKkdBA" title="Generating Playwright tests in VS Code" />
-   **錄製新測試**：點選側邊欄中的**錄製新測試**。瀏覽器視窗會開啟。當您與頁面互動時，Playwright 會自動產生測試程式碼。您也可以從錄製工具列產生斷言。

<ProgressiveImage image={require("../images/getting-started/record-new-test.png")} alt="record a new test" />
-   **在游標處錄製**：將游標放在現有測試內，然後點選**在游標處錄製**，即可在該特定位置新增新動作。

<ProgressiveImage image={require("../images/getting-started/record-at-cursor.png")} alt="record at cursor" />
-   **選取定位器**：使用**選取定位器**工具來點選開啟的瀏覽器中的任何元素。Playwright 會決定最佳的定位器並將其複製到您的剪貼簿，準備貼到您的程式碼中。

<ProgressiveImage image={require("../images/getting-started/pick-locator.png")} alt="pick locators" />

若要了解更多資訊，請參閱我們的 [CodeGen 指南](./codegen.mdx)。

## 進階功能

### 專案相依套件

使用[專案相依套件](./test-projects.mdx)來定義在其他測試之前執行的設定測試。例如，您可以建立一個首先執行的登入測試，然後在多個測試中重複使用該已驗證狀態，而無需為每個測試重新登入。在 VS Code 中，您可以在測試檔案總管中看到這些設定測試，並在需要時獨立執行它們。

<ProgressiveImage image={require("../images/getting-started/setup-tests.png")} alt="setup tests in vscode" />

若要了解更多資訊，請參閱我們的[專案相依套件指南](./test-projects.mdx)。

### 全域設定

對於需要在所有測試之前只執行一次的任務（如播種資料庫），請使用**全域設定**。您可以從 Playwright 側邊欄手動觸發全域設定和拆解。

<ProgressiveImage image={require("../images/getting-started/global-setup.png")} alt="running global setup" />

### 多重設定配置

如果您有多個 `playwright.config.ts` 檔案，您可以使用 Playwright 側邊欄中的齒輪圖示在它們之間切換。這讓您可以輕鬆使用不同的測試套件或環境。

<ProgressiveImage image={require("../images/getting-started/selecting-configuration.png")} alt="Selecting a configuration file" />

## 快速參考

| 動作                    | 在 VS Code 中的操作方式                                     |
| ----------------------- | ----------------------------------------------------------- |
| **安裝 Playwright**     | 命令選擇區 → `Test: Install Playwright`                     |
| **執行測試**            | 點選測試旁邊的「播放」圖示                                   |
| **偵錯測試**            | 設定中斷點，右鍵點選測試 → `Debug Test`                     |
| **顯示即時瀏覽器**      | 在 Playwright 側邊欄中啟用 `Show Browsers`                  |
| **錄製新測試**          | 點選 Playwright 側邊欄中的 `Record new`                     |
| **選取定位器**          | 點選 Playwright 側邊欄中的 `Pick locator`                   |
| **檢視測試追蹤**        | 在 Playwright 側邊欄中啟用 `Show Trace Viewer`              |
