## 簡介

Playwright 具備為您產生測試的能力，當您在瀏覽器中執行操作時，它會自動錄製測試，這是快速開始測試的絕佳方式。Playwright 會檢視您的頁面並找出最佳的定位器，優先使用[角色、文字與測試 ID 定位器](./locators.mdx)。如果產生器發現多個元素符合定位器，它會改善定位器使其更穩固，以唯一識別目標元素。

## 在 VS Code 中產生測試

安裝 VS Code 擴充套件並直接從 VS Code 產生測試。該擴充套件可在 [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=ms-playwright.playwright) 中取得。請參閱我們的 [VS Code 入門](./getting-started-vscode.mdx) 指南。

<LiteYouTube id="LM4yqrOzmFE" title="Generating Playwright tests in VS Code" />

### 錄製新測試

要錄製測試，請從 Testing 側邊欄點擊 **Record new** 按鈕。這會建立一個 `test-1.spec.ts` 檔案並開啟瀏覽器視窗。

<img width="1385" alt="record new in vs code" src="https://user-images.githubusercontent.com/13063165/220961665-615d0ab8-3f0b-439c-ad0b-0424d9aa154b.png" />

在瀏覽器中前往您想要測試的 URL，並開始點擊以錄製您的使用者操作。

![generating user actions](https://github.com/microsoft/playwright/assets/13063165/1d4c8f37-8325-4816-a665-d0e95e63f509)

Playwright 會錄製您的操作並直接在 VS Code 中產生測試程式碼。您也可以透過選擇工具列中的其中一個圖示，然後點擊頁面上的元素來產生斷言。可以產生以下斷言：
* `'assert visibility'` 斷言元素是可見的
* `'assert text'` 斷言元素包含特定文字
* `'assert value'` 斷言元素具有特定值

![generating assertions](https://github.com/microsoft/playwright/assets/13063165/d131eb35-b2ca-4bf4-a8ac-88b6e40dcf07)

錄製完成後，點擊 **cancel** 按鈕或關閉瀏覽器視窗。然後您可以檢查 `test-1.spec.ts` 檔案，並在需要時手動改進它。

![code from a generated test](https://github.com/microsoft/playwright/assets/13063165/2ba4c212-4713-460a-b054-6dc6b67a9a7c)

### 在游標處錄製

要從測試中的特定點開始錄製，請將游標移動到您想要錄製更多操作的位置，然後從 Testing 側邊欄點擊 **Record at cursor** 按鈕。如果您的瀏覽器視窗尚未開啟，請先勾選 'Show browser' 執行測試，然後點擊 **Record at cursor** 按鈕。

![record at cursor in vs code](https://github.com/microsoft/playwright/assets/13063165/77948ab8-92a2-435f-9833-0944da5ae664)

在瀏覽器視窗中開始執行您想要錄製的操作。

<img width="1394" alt="add feed the dog to todo app" src="https://user-images.githubusercontent.com/13063165/220960770-6435cec7-1723-42a8-8c1f-8244e2d800c7.png" />

在 VS Code 的測試檔案中，您會看到新產生的操作已添加到游標位置的測試中。

![code from a generated test](https://github.com/microsoft/playwright/assets/13063165/4f4bb34e-9cda-41fe-bf65-8d8016d84c7f)

### 產生定位器

您可以使用測試產生器產生定位器。
- 從 Testing 側邊欄點擊 **Pick locator** 按鈕，然後將滑鼠游標移到瀏覽器視窗中的元素上，以查看每個元素下方突出顯示的[定位器](./locators.mdx)。
- 點擊您需要的元素，它現在會顯示在 VS Code 的 **Pick locator** 方塊中。
- 在鍵盤上按 <kbd>Enter</kbd> 將定位器複製到剪貼簿，然後貼到程式碼中的任何位置。或者按 'escape' 取消。

<img width="1641" alt="Pick locators in VS code" src="https://user-images.githubusercontent.com/13063165/220958368-95b03620-3c9b-40a8-be74-01c96ba03cad.png" />

## 使用 Playwright Inspector 產生測試

執行 `codegen` 指令時會開啟兩個視窗，一個是瀏覽器視窗，您可以在其中與要測試的網站互動，另一個是 Playwright Inspector 視窗，您可以在其中錄製測試並將其複製到編輯器中。

### 執行 Codegen

使用 `codegen` 指令執行測試產生器，後面接您要為其產生測試的網站 URL。URL 是可選的，您也可以不使用它執行指令，然後直接在瀏覽器視窗中加入 URL。

```bash
npx playwright codegen demo.playwright.dev/todomvc
```

### 錄製測試

執行 `codegen` 指令並在瀏覽器視窗中執行操作。Playwright 會為使用者互動產生程式碼，您可以在 Playwright Inspector 視窗中看到。完成測試錄製後，停止錄製並按 **copy** 按鈕將產生的測試複製到編輯器中。

使用測試產生器您可以錄製：
* 操作，如點擊或填寫，只需與頁面互動即可
* 斷言，透過點擊工具列中的其中一個圖示，然後點擊頁面上的元素進行斷言。您可以選擇：
  * `'assert visibility'` 斷言元素是可見的
  * `'assert text'` 斷言元素包含特定文字
  * `'assert value'` 斷言元素具有特定值

###### 
![Recording a test](https://github.com/microsoft/playwright/assets/13063165/34a79ea1-639e-4cb3-8115-bfdc78e3d34d)

###### 
當您完成與頁面的互動後，按 **record** 按鈕停止錄製，並使用 **copy** 按鈕將產生的程式碼複製到編輯器中。

使用 **clear** 按鈕清除程式碼以重新開始錄製。完成後，關閉 Playwright Inspector 視窗或停止終端指令。

### 產生定位器

您可以使用測試產生器產生[定位器](/locators.mdx)。
* 按 `'Record'` 按鈕停止錄製，`'Pick Locator'` 按鈕會出現。
* 點擊 `'Pick Locator'` 按鈕，然後將滑鼠游標移到瀏覽器視窗中的元素上，以查看每個元素下方突出顯示的定位器。
* 要選擇定位器，請點擊您想要定位的元素，該定位器的程式碼會出現在 Pick Locator 按鈕旁邊的欄位中。
* 然後您可以在此欄位中編輯定位器以進行微調，或使用複製按鈕將其複製並貼到程式碼中。

###### 
![picking a locator](https://github.com/microsoft/playwright/assets/13063165/2c8a12e2-4e98-4fdd-af92-1d73ae696d86)

## 模擬

您可以使用測試產生器透過模擬來產生測試，以便為特定檢視區、裝置、色彩配置產生測試，以及模擬地理位置、語言或時區。測試產生器也可以在保留已驗證狀態的同時產生測試。

### 模擬檢視區大小

Playwright 開啟瀏覽器視窗時會將其檢視區設定為特定的寬度和高度，並且不是響應式的，因為測試需要在相同條件下執行。使用 `--viewport` 選項以不同的檢視區大小產生測試。

```bash
npx playwright codegen --viewport-size="800,600" playwright.dev
```

###### 
<img width="870" alt="Codegen generating code for tests for playwright.dev website with a specific viewport js" src="https://user-images.githubusercontent.com/13063165/220402029-f90d1c9f-d740-4c0f-acc8-95235ee83f85.png" />

### 模擬裝置

使用 `--device` 選項在模擬行動裝置時錄製腳本和測試，該選項會設定檢視區大小和 user agent 等。

```bash
npx playwright codegen --device="iPhone 13" playwright.dev
```

###### 
<img width="1300" alt="Codegen generating code for tests for playwright.dev website emulated for iPhone 13 js" src="https://user-images.githubusercontent.com/13063165/220921482-dc4f5532-9dce-40bd-8a28-e0d87d26a601.png" />

### 模擬色彩配置

使用 `--color-scheme` 選項在模擬色彩配置時錄製腳本和測試。

```bash
npx playwright codegen --color-scheme=dark playwright.dev
```

###### 
<img width="1394" alt="Codegen generating code for tests for playwright.dev website in dark mode js" src="https://user-images.githubusercontent.com/13063165/220930273-f3a25bae-64dd-4bbb-99ed-1e97c0cb1ebf.png" />

### 模擬地理位置、語言和時區

使用 `--timezone`、`--geolocation` 和 `--lang` 選項在模擬時區、語言和位置時錄製腳本和測試。頁面開啟後：
1. 接受 cookie
1. 在右上角，點擊定位我按鈕以查看地理位置的實際運作。

```bash
npx playwright codegen --timezone="Europe/Rome" --geolocation="41.890221,12.492348" --lang="it-IT" bing.com/maps
```

###### 
<img width="1394" alt="Codegen generating code for tests for bing maps showing timezone, geolocation as Rome, Italy and in Italian language" src="https://user-images.githubusercontent.com/13063165/220931996-d3144421-8d3b-4f9f-896c-769c01566c01.png" />

### 保留已驗證狀態

使用 `--save-storage` 執行 `codegen` 以在會話結束時儲存 [cookies](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)、[localStorage](https://developer.mozilla.org/en-US/docs/Web/API/Window/localStorage) 和 [IndexedDB](https://developer.mozilla.org/en-US/docs/Web/API/IndexedDB_API) 資料。這對於單獨錄製驗證步驟並在稍後錄製更多測試時重複使用很有用。

```bash
npx playwright codegen github.com/microsoft/playwright --save-storage=auth.json
```

###### 
<img width="1394" alt="github page before logging in js" src="https://user-images.githubusercontent.com/13063165/220929062-88dfe567-0c6d-4e49-b9f9-74ae241fb8c7.png" />

#### 登入

執行驗證並關閉瀏覽器後，`auth.json` 會包含儲存狀態，您可以在測試中重複使用。

<img width="1394" alt="login to GitHub screen" src="https://user-images.githubusercontent.com/13063165/220561688-04b2b984-4ba6-4446-8b0a-8058876e2a02.png" />

請確保您僅在本機使用 `auth.json`，因為它包含敏感資訊。將其加入 `.gitignore` 或在完成測試產生後刪除它。

#### 載入已驗證狀態

使用 `--load-storage` 執行以從 `auth.json` 載入先前載入的儲存。這樣，所有 [cookies](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)、[localStorage](https://developer.mozilla.org/en-US/docs/Web/API/Window/localStorage) 和 [IndexedDB](https://developer.mozilla.org/en-US/docs/Web/API/IndexedDB_API) 資料都會被還原，使大多數 Web 應用程式進入已驗證狀態，而無需再次登入。這意味著您可以從已登入狀態繼續產生測試。

```bash
npx playwright codegen --load-storage=auth.json github.com/microsoft/playwright
```

###### 
<img width="1394" alt="github signed in showing use of load storage js" src="https://user-images.githubusercontent.com/13063165/220927873-9e55fdda-2def-45c1-9a1b-bcc851885f96.png" />

#### 使用現有的 userDataDir

使用 `--user-data-dir` 執行 `codegen` 為瀏覽器會話設定固定的[使用者資料目錄](https://playwright.dev/docs/api/class-browsertype#browser-type-launch-persistent-context-option-user-data-dir)。如果您建立自訂瀏覽器使用者資料目錄，codegen 會使用此現有瀏覽器設定檔，並可存取該設定檔中存在的任何驗證狀態。

:::warning

[從 Chrome 136 開始，預設使用者資料目錄無法透過自動化工具存取](https://developer.chrome.com/blog/remote-debugging-port)，例如 Playwright。您必須建立單獨的使用者資料目錄以供測試使用。
:::

```bash
npx playwright codegen --user-data-dir=/path/to/your/browser/data/ github.com/microsoft/playwright
```

## 使用自訂設定進行錄製

如果您想在某些非標準設定中使用 codegen（例如，使用 [browserContext.route()](/api/class-browsercontext.mdx#browser-context-route)），可以呼叫 [page.pause()](/api/class-page.mdx#page-pause)，這會開啟一個帶有 codegen 控制項的單獨視窗。

```js
const { chromium } = require('@playwright/test');

(async () => {
  // Make sure to run headed.
  const browser = await chromium.launch({ headless: false });

  // Setup context however you like.
  const context = await browser.newContext({ /* pass any options */ });
  await context.route('**/*', route => route.continue());

  // Pause the page, and start recording manually.
  const page = await context.newPage();
  await page.pause();
})();
```
