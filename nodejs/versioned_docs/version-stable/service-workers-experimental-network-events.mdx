## 簡介

:::warning

如果您正在尋找一般的網路模擬、路由和攔截，請先參閱 [Network Guide](./network.mdx)。Playwright 提供內建的 API 用於此用途，不需要下面的資訊。然而，如果您對 Service Workers 自身發出的請求感興趣，請繼續閱讀。
:::

[Service Workers](https://developer.mozilla.org/en-US/docs/Web/API/Service_Worker_API) 提供瀏覽器原生方法來處理頁面使用原生 [Fetch API (`fetch`)](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API) 發出的請求，以及其他網路請求的資源（如腳本、CSS 和圖片）。

它們可以作為頁面與外部網路之間的**網路代理**來執行快取邏輯，或者如果 Service Worker 新增了 [FetchEvent](https://developer.mozilla.org/en-US/docs/Web/API/FetchEvent#examples) 監聽器，可以為使用者提供離線體驗。

許多使用 Service Workers 的網站僅將它們用作透明的最佳化技術。雖然使用者可能會注意到更快的體驗，但應用程式的實作並不知道它們的存在。啟用或不啟用 Service Workers 運行應用程式在功能上顯得等同。

## 如何啟用

Playwright 對 Service Workers 發出的請求的檢查和路由是**實驗性的**，預設為停用。

將 `PW_EXPERIMENTAL_SERVICE_WORKER_NETWORK_EVENTS` 環境變數設定為 `1`（或任何其他值）以啟用此功能。目前僅支援 Chrome/Chromium。

如果您正在使用（或有興趣使用此功能），請在[此問題](https://github.com/microsoft/playwright/issues/15684)上留言，告訴我們您的使用案例。

## Service Worker 抓取

### 存取 Service Workers 和等待啟動

您可以使用 [browserContext.serviceWorkers()](/api/class-browsercontext.mdx#browser-context-service-workers) 來列出 Service [Worker]，或者如果您預期頁面會觸發其[註冊](https://developer.mozilla.org/en-US/docs/Web/API/ServiceWorkerContainer/register)，可以特別監視 Service [Worker]：

```js
const serviceWorkerPromise = context.waitForEvent('serviceworker');
await page.goto('/example-with-a-service-worker.html');
const serviceworker = await serviceWorkerPromise;
```

[browserContext.on('serviceworker')](/api/class-browsercontext.mdx#browser-context-event-service-worker) 在 Service Worker 的主腳本被求值**之前**觸發，因此在呼叫 service[worker.evaluate()](/api/class-worker.mdx#worker-evaluate) **之前**，您應該等待其啟動。

等待 Service Worker 啟動有更慣用的方法，但以下是與實作無關的方法：

```js
await page.evaluate(async () => {
  const registration = await window.navigator.serviceWorker.getRegistration();
  if (registration.active?.state === 'activated')
    return;
  await new Promise(res =>
    window.navigator.serviceWorker.addEventListener('controllerchange', res),
  );
});
```

### 網路事件和路由

由 **Service Worker** 發出的任何網路請求都會有：
* [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) 和其對應的事件（[browserContext.on('requestfinished')](/api/class-browsercontext.mdx#browser-context-event-request-finished) 和 [browserContext.on('response')](/api/class-browsercontext.mdx#browser-context-event-response)，或 [browserContext.on('requestfailed')](/api/class-browsercontext.mdx#browser-context-event-request-failed)）
* [browserContext.route()](/api/class-browsercontext.mdx#browser-context-route) 將會看到請求
* [request.serviceWorker()](/api/class-request.mdx#request-service-worker) 將會設定為 Service [Worker] 實例，而 [request.frame()](/api/class-request.mdx#request-frame) 將會**拋出例外**
* [response.fromServiceWorker()](/api/class-response.mdx#response-from-service-worker) 將會回傳 `false`

此外，由 **Page**（包括其子 [Frame]）發出的任何網路請求都會有：
* [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) 和其對應的事件（[browserContext.on('requestfinished')](/api/class-browsercontext.mdx#browser-context-event-request-finished) 和 [browserContext.on('response')](/api/class-browsercontext.mdx#browser-context-event-response)，或 [browserContext.on('requestfailed')](/api/class-browsercontext.mdx#browser-context-event-request-failed)）
* [page.on('request')](/api/class-page.mdx#page-event-request) 和其對應的事件（[page.on('requestfinished')](/api/class-page.mdx#page-event-request-finished) 和 [page.on('response')](/api/class-page.mdx#page-event-response)，或 [page.on('requestfailed')](/api/class-page.mdx#page-event-request-failed)）
* [page.route()](/api/class-page.mdx#page-route) 和 [page.route()](/api/class-page.mdx#page-route) 將**不會**看到請求（如果 Service Worker 的 fetch 處理器已註冊）
* [request.serviceWorker()](/api/class-request.mdx#request-service-worker) 將會設定為 `null`，而 [request.frame()](/api/class-request.mdx#request-frame) 將會回傳 [Frame]
* [response.fromServiceWorker()](/api/class-response.mdx#response-from-service-worker) 將會回傳 `true`（如果 Service Worker 的 fetch 處理器已註冊）

許多 Service Worker 實作只是簡單地執行來自頁面的請求（可能會省略一些自訂快取/離線邏輯以簡化）：

```js title="transparent-service-worker.js"
self.addEventListener('fetch', event => {
  // actually make the request
  const responsePromise = fetch(event.request);
  // send it back to the page
  event.respondWith(responsePromise);
});

self.addEventListener('activate', event => {
  event.waitUntil(clients.claim());
});
```

如果頁面註冊了上述 Service Worker：

```html
<!-- filename: index.html -->
<script>
  window.registrationPromise = navigator.serviceWorker.register('/transparent-service-worker.js');
</script>
```

在透過 [page.goto()](/api/class-page.mdx#page-goto) 首次造訪頁面時，將會發出以下 Request/Response 事件（以及對應的網路生命週期事件）：

| Event                             | Owner            | URL                            | Routed | [response.fromServiceWorker()](/api/class-response.mdx#response-from-service-worker) |
| -                                 | -                | -                              | -      | -                                      |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | index.html                     | Yes    |                                        |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | index.html                     | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | Service [Worker] | transparent-service-worker.js  | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | Service [Worker] | data.json                      | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | data.json                      |        | Yes                                    |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | data.json                      |        | Yes                                    |

由於範例 Service Worker 只是作為基本的透明「代理」：
* `data.json` 有 2 個 [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) 事件；一個是 [Frame] 擁有的，另一個是 Service [Worker] 擁有的。
* 只有 Service [Worker] 擁有的資源請求可以透過 [browserContext.route()](/api/class-browsercontext.mdx#browser-context-route) 路由；`data.json` 的 [Frame] 擁有事件不可路由，因為它們甚至沒有可能撞到外部網路，因為 Service Worker 已註冊了 fetch 處理器。

:::caution

重要提醒：如果對具有非 null [request.serviceWorker()](/api/class-request.mdx#request-service-worker) 的 [Request]/[Response] 呼叫 [request.frame()](/api/class-request.mdx#request-frame) 或 [response.frame()](/api/class-response.mdx#response-frame)，將會**拋出例外**。
:::

#### 進階範例

當 Service Worker 處理頁面的請求時，Service Worker 可以向外部網路發出 0 到 n 個請求。Service Worker 可能直接從快取回應、在記憶體中產生回應、重寫請求、發出兩個請求然後合併為 1 個等。

考慮下面的程式碼片段來了解 Playwright 對 Request/Responses 的視圖以及它在這些情況下如何影響路由。

```js title="complex-service-worker.js"
self.addEventListener('install', function(event) {
  event.waitUntil(
      caches.open('v1').then(function(cache) {
      // 1. Pre-fetches and caches /addressbook.json
        return cache.add('/addressbook.json');
      })
  );
});

// Opt to handle FetchEvent's from the page
self.addEventListener('fetch', event => {
  event.respondWith(
      (async () => {
        // 1. Try to first serve directly from caches
        const response = await caches.match(event.request);
        if (response)
          return response;

        // 2. Re-write request for /foo to /bar
        if (event.request.url.endsWith('foo'))
          return fetch('./bar');

        // 3. Prevent tracker.js from being retrieved, and returns a placeholder response
        if (event.request.url.endsWith('tracker.js')) {
          return new Response('console.log("no trackers!")', {
            status: 200,
            headers: { 'Content-Type': 'text/javascript' },
          });
        }

        // 4. Otherwise, fallthrough, perform the fetch and respond
        return fetch(event.request);
      })()
  );
});

self.addEventListener('activate', event => {
  event.waitUntil(clients.claim());
});
```

以及一個簡單註冊 Service Worker 的頁面：

```html
<!-- filename: index.html -->
<script>
  window.registrationPromise = navigator.serviceWorker.register('/complex-service-worker.js');
</script>
```

在透過 [page.goto()](/api/class-page.mdx#page-goto) 首次造訪頁面時，將會發出以下 Request/Response 事件：

| Event                             | Owner            | URL                            | Routed | [response.fromServiceWorker()](/api/class-response.mdx#response-from-service-worker) |
| -                                 | -                | -                              | -      | -                                      |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | index.html                     | Yes    |                                        |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | index.html                     | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | Service [Worker] | complex-service-worker.js      | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | Service [Worker] | addressbook.json               | Yes    |                                        |

重要提醒，[`cache.add`](https://developer.mozilla.org/en-US/docs/Web/API/Cache/add) 使 Service Worker 發出了請求（Service [Worker] 擁有的），甚至在頁面要求 `addressbook.json` 之前。

一旦 Service Worker 啟動並處理 FetchEvents，如果頁面發出以下請求：

```js
await page.evaluate(() => fetch('/addressbook.json'));
await page.evaluate(() => fetch('/foo'));
await page.evaluate(() => fetch('/tracker.js'));
await page.evaluate(() => fetch('/fallthrough.txt'));
```

將會發出以下 Request/Response 事件：

| Event                             | Owner            | URL                            | Routed | [response.fromServiceWorker()](/api/class-response.mdx#response-from-service-worker) |
| -                                 | -                | -                              | -      | -                                      |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | addressbook.json               |        | Yes                                    |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | addressbook.json               |        | Yes                                    |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | Service [Worker] | bar                            | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | foo                            |        | Yes                                    |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | foo                            |        | Yes                                    |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | tracker.js                     |        | Yes                                    |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | tracker.js                     |        | Yes                                    |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | Service [Worker] | fallthrough.txt                | Yes    |                                        |
| [browserContext.on('request')](/api/class-browsercontext.mdx#browser-context-event-request) | [Frame]          | fallthrough.txt                |        | Yes                                    |
| [page.on('request')](/api/class-page.mdx#page-event-request)           | [Frame]          | fallthrough.txt                |        | Yes                                    |

重要提醒：
* 頁面請求了 `/foo`，但 Service Worker 請求了 `/bar`，所以只有 `/foo` 的 [Frame] 擁有事件，而沒有 `/bar` 的。
* 同樣地，Service Worker 從未為 `tracker.js` 撞到網路，所以只為該請求發出了 [Frame] 擁有的事件。

## 僅路由 Service Worker 請求

```js
await context.route('**', async route => {
  if (route.request().serviceWorker()) {
    // NB: calling route.request().frame() here would THROW
    return route.fulfill({
      contentType: 'text/plain',
      status: 200,
      body: 'from sw',
    });
  } else {
    return route.continue();
  }
});
```

## 已知限制

目前無法路由更新的 Service Worker 主腳本程式碼的請求 (https://github.com/microsoft/playwright/issues/14711)。
