## 簡介

Node.js 的 Playwright 有一個 canary 發佈系統。

它允許你**測試新的未發布功能**，而不是等待完整發布。它們每天都會在 Playwright 的 `next` NPM 標籤上發布。

這是一個**給維護者反饋**的好方法，確保新實現的功能按預期工作。

:::note
 使用金絲雀版本在生產環境中可能看起來很冒險，但實際上並不是這樣。金絲雀版本通過了所有自動化測試，並用於測試例如 HTML 報告、Trace Viewer 或 Playwright Inspector 與端到端測試。
:::

```txt
npm install -D @playwright/test@next
```

## Next npm Dist Tag

對於 `main` 上的任何程式碼相關提交，持續整合將在 `@next` npm 發佈標籤下發佈每日金絲雀版本。

你可以在 [npm](https://www.npmjs.com/package/@playwright/test?activeTab=versions) 上看到當前的 dist 標籤:

- `latest`: 穩定版本
- `next`: 下一個版本，每日發布
- `beta`: 在發布分支被切割後，通常在穩定版本發布前一週，每次提交都會在此標籤下發布

## 使用金絲雀發布

```txt
npm install -D @playwright/test@next
```

## 文件

穩定版和 `next` 文件已發布在 [playwright.dev](https://playwright.dev)。要查看 `next` 文件，請按鍵盤上的 <kbd>Shift</kbd> 鍵 `5` 次。
