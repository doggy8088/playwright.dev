此 API 用於 Web API 測試。您可以使用它來觸發 API 端點、組態設定微服務、準備環境或服務以進行端對端測試。

每個 Playwright 瀏覽器情境都有一個關聯的 [APIRequestContext] 實例，它與瀏覽器情境共享 cookie 儲存，並可透過 [browserContext.request](/api/class-browsercontext.mdx#browser-context-request) 或 [page.request](/api/class-page.mdx#page-request) 存取。也可以透過呼叫 [apiRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context) 手動建立新的 `APIRequestContext` 實例。

**Cookie 管理**

由 [browserContext.request](/api/class-browsercontext.mdx#browser-context-request) 和 [page.request](/api/class-page.mdx#page-request) 傳回的 [APIRequestContext] 與對應的 [BrowserContext] 共享 cookie 儲存。每個 API 請求都會有 `Cookie` 標頭，填入來自瀏覽器情境的值。如果 API 回應包含 `Set-Cookie` 標頭，它會自動更新 [BrowserContext] cookies，並且從頁面發出的請求會取得它們。這意味著如果您使用此 API 登入，您的端對端測試將會登入，反之亦然。

如果您想要 API 請求不干擾瀏覽器 cookies，您應該透過呼叫 [apiRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context) 建立新的 [APIRequestContext]。這樣的 `APIRequestContext` 物件將擁有自己的隔離 cookie 儲存。

### delete {#api-request-context-delete}

<font size="2" style={{position: "relative", top: "-20px"}}>新增於：v1.16</font><x-search>apiRequestContext.delete</x-search>

傳送 HTTP(S) [DELETE](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/DELETE) 請求並傳回其回應。此方法會從情境填入請求 cookies，並從回應更新情境 cookies。此方法會自動跟隨重新導向。

**使用方式**

```js
await apiRequestContext.delete(url);
await apiRequestContext.delete(url, options);
```

**參數**
- `url` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-url"/><a href="#api-request-context-delete-option-url" class="list-anchor">#</a>
  
  目標 URL。
- `options` [Object] *(選用)*
  - `data` [string] | [Buffer] | [Serializable] *(選用)* <font size="2">新增於：v1.17</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-data"/><a href="#api-request-context-delete-option-data" class="list-anchor">#</a>
    
    允許設定請求的 post 資料。如果 data 參數是物件，它會被序列化為 json 字串，並且如果未明確設定，`content-type` 標頭會設定為 `application/json`。否則如果未明確設定，`content-type` 標頭會設定為 `application/octet-stream`。
  - `failOnStatusCode` [boolean] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-fail-on-status-code"/><a href="#api-request-context-delete-option-fail-on-status-code" class="list-anchor">#</a>

  - `form` [Object]&lt;[string], [string] | [number] | [boolean]&gt; | [FormData] *(選用)* <font size="2">新增於：v1.17</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-form"/><a href="#api-request-context-delete-option-form" class="list-anchor">#</a>
    
    提供一個物件，該物件會使用 `application/x-www-form-urlencoded` 編碼序列化為 html 表單，並作為此請求主體傳送。如果指定此參數，除非明確提供，否則 `content-type` 標頭會設定為 `application/x-www-form-urlencoded`。

  - `ignoreHTTPSErrors` [boolean] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-ignore-https-errors"/><a href="#api-request-context-delete-option-ignore-https-errors" class="list-anchor">#</a>
    
    傳送網路請求時是否忽略 HTTPS 錯誤。預設為 `false`。
  - `maxRedirects` [number] *(選用)* <font size="2">新增於：v1.26</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-max-redirects"/><a href="#api-request-context-delete-option-max-redirects" class="list-anchor">#</a>

  - `maxRetries` [number] *(選用)* <font size="2">新增於：v1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-max-retries"/><a href="#api-request-context-delete-option-max-retries" class="list-anchor">#</a>
    
    網路錯誤應重試的最大次數。目前僅重試 `ECONNRESET` 錯誤。不會根據 HTTP 回應碼重試。如果超出限制將拋出錯誤。預設為 `0` - 不重試。
  - `multipart` [FormData] | [Object]&lt;[string], [string] | [number] | [boolean] | [ReadStream] | [Object]&gt; *(選用)* <font size="2">新增於：v1.17</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-delete-option-multipart"/><a href="#api-request-context-delete-option-multipart" class="list-anchor">#</a>

### dispose {#api-request-context-dispose}

<font size="2" style={{position: "relative", top: "-20px"}}>新增於：v1.16</font><x-search>apiRequestContext.dispose</x-search>

由 [apiRequestContext.get()](/api/class-apirequestcontext.mdx#api-request-context-get) 和類似方法傳回的所有回應都儲存在記憶體中，因此您可以稍後呼叫 [apiResponse.body()](/api/class-apiresponse.mdx#api-response-body)。此方法會丟棄其所有資源，對已釋放的 [APIRequestContext] 呼叫任何方法都會拋出例外。

**使用方式**

```js
await apiRequestContext.dispose();
await apiRequestContext.dispose(options);
```

- `options` [Object] *(選用)*
  - `reason` [string] *(選用)* <font size="2">新增於：v1.45</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-dispose-option-reason"/><a href="#api-request-context-dispose-option-reason" class="list-anchor">#</a>
    
    由情境釋放中斷的操作要回報的原因。

### fetch {#api-request-context-fetch}

<font size="2" style={{position: "relative", top: "-20px"}}>新增於：v1.16</font><x-search>apiRequestContext.fetch</x-search>

**使用方式**

JSON 物件可以直接傳遞給請求：

```js

  method: 'post',
  data: {
    title: 'Book Title',
    author: 'John Doe',
  }
});
```

在請求主體中傳送檔案的常見方式是透過指定 `multipart` 參數，將它們作為 `multipart/form-data` 編碼的表單欄位上傳：

```js
const form = new FormData();
form.set('name', 'John');
form.append('name', 'Doe');
// 傳送兩個同名的檔案欄位。
form.append('file', new File(['console.log(2024);'], 'f1.js', { type: 'text/javascript' }));
form.append('file', new File(['hello'], 'f2.txt', { type: 'text/plain' }));
await request.fetch('https://example.com/api/uploadForm', {
  multipart: form
});
```

### get {#api-request-context-get}

<font size="2" style={{position: "relative", top: "-20px"}}>新增於：v1.16</font><x-search>apiRequestContext.get</x-search>

// 以物件形式傳遞參數
await request.get('https://example.com/api/getText', {
  params: {
    'isbn': '1234',
    'page': 23,
  }
});

// 以 URLSearchParams 形式傳遞參數
const searchParams = new URLSearchParams();
searchParams.set('isbn', '1234');
searchParams.append('page', 23);
searchParams.append('page', 24);
await request.get('https://example.com/api/getText', { params: searchParams });

// 以字串形式傳遞參數
const queryString = 'isbn=1234&page=23&page=24';
await request.get('https://example.com/api/getText', { params: queryString });

  - `data` [string] | [Buffer] | [Serializable] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-data"/><a href="#api-request-context-put-option-data" class="list-anchor">#</a>
    
    允許設定請求的 post 資料。如果 data 參數是物件，它會被序列化為 json 字串，並且如果未明確設定，`content-type` 標頭會設定為 `application/json`。否則如果未明確設定，`content-type` 標頭會設定為 `application/octet-stream`。
  - `failOnStatusCode` [boolean] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-fail-on-status-code"/><a href="#api-request-context-put-option-fail-on-status-code" class="list-anchor">#</a>

  - `headers` [Object]&lt;[string], [string]&gt; *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-headers"/><a href="#api-request-context-put-option-headers" class="list-anchor">#</a>

  - `ignoreHTTPSErrors` [boolean] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-ignore-https-errors"/><a href="#api-request-context-put-option-ignore-https-errors" class="list-anchor">#</a>
    
    傳送網路請求時是否忽略 HTTPS 錯誤。預設為 `false`。
  - `maxRedirects` [number] *(選用)* <font size="2">新增於：v1.26</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-max-redirects"/><a href="#api-request-context-put-option-max-redirects" class="list-anchor">#</a>

  - `maxRetries` [number] *(選用)* <font size="2">新增於：v1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-max-retries"/><a href="#api-request-context-put-option-max-retries" class="list-anchor">#</a>
    
    網路錯誤應重試的最大次數。目前僅重試 `ECONNRESET` 錯誤。不會根據 HTTP 回應碼重試。如果超出限制將拋出錯誤。預設為 `0` - 不重試。

      檔案名稱
    - `mimeType` [string]
      
      檔案類型
    - `buffer` [Buffer]
      
      檔案內容

  - `params` [Object]&lt;[string], [string] | [number] | [boolean]&gt; | [URLSearchParams] | [string] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-params"/><a href="#api-request-context-put-option-params" class="list-anchor">#</a>
    
    要與 URL 一起傳送的查詢參數。
  - `timeout` [number] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-timeout"/><a href="#api-request-context-put-option-timeout" class="list-anchor">#</a>
    
    請求逾時時間，以毫秒為單位。預設為 `30000`（30 秒）。傳遞 `0` 表示停用逾時。

### storageState {#api-request-context-storage-state}

<font size="2" style={{position: "relative", top: "-20px"}}>新增於：v1.16</font><x-search>apiRequestContext.storageState</x-search>

傳回此請求情境的儲存狀態，包含目前的 cookies 和本機儲存快照（如果已傳遞給建構函式）。

**使用方式**

```js
await apiRequestContext.storageState();
await apiRequestContext.storageState(options);
```

**參數**
- `options` [Object] *(選用)*
  - `indexedDB` [boolean] *(選用)* <font size="2">新增於：v1.51</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-option-indexed-db"/><a href="#api-request-context-storage-state-option-indexed-db" class="list-anchor">#</a>
    
    設定為 `true` 以在儲存狀態快照中包含 IndexedDB。
  - `path` [string] *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-option-path"/><a href="#api-request-context-storage-state-option-path" class="list-anchor">#</a>
    
    儲存儲存狀態的檔案路徑。如果 [path](/api/class-apirequestcontext.mdx#api-request-context-storage-state-option-path) 是相對路徑，則會相對於目前工作目錄解析。如果未提供路徑，仍會傳回儲存狀態，但不會儲存到磁碟。

**傳回值**

    - `origin` [string]

      - `value` [string]
