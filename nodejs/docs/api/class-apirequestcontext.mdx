此 API 用於 Web API 測試。您可以使用它來觸發 API 端點、設定微服務、準備環境或為您的端對端測試準備服務。

每個 Playwright 瀏覽器情境都有與其關聯的 [APIRequestContext] 實例，該實例與瀏覽器情境共享 cookie 儲存，並可透過 [browserContext.request](/api/class-browsercontext.mdx#browser-context-request) 或 [page.request](/api/class-page.mdx#page-request) 存取。也可以透過呼叫 [apiRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context) 手動建立新的 APIRequestContext 實例。

**Cookie 管理**

由 [browserContext.request](/api/class-browsercontext.mdx#browser-context-request) 和 [page.request](/api/class-page.mdx#page-request) 回傳的 [APIRequestContext] 與對應的 [BrowserContext] 共享 cookie 儲存。每個 API 請求都會使用來自瀏覽器情境的值填入 `Cookie` 標頭。如果 API 回應包含 `Set-Cookie` 標頭，它會自動更新 [BrowserContext] cookie，並且從頁面發出的請求會選取它們。這意味著如果您使用此 API 登入，您的端對端測試將會被登入，反之亦然。

如果您希望 API 請求不干擾瀏覽器 cookie，您應該透過呼叫 [apiRequest.newContext()](/api/class-apirequest.mdx#api-request-new-context) 建立新的 [APIRequestContext]。這樣的 `APIRequestContext` 物件將擁有自己的隔離 cookie 儲存。

### delete {#api-request-context-delete}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.delete</x-search>

傳送 HTTP(S) [DELETE](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/DELETE) 請求並回傳其回應。此方法會從情境填入請求 cookie，並從回應更新情境 cookie。此方法會自動追蹤重新導向。

```js
await apiRequestContext.delete(url);
await apiRequestContext.delete(url, options);
```

### dispose {#api-request-context-dispose}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.dispose</x-search>

All responses returned by [apiRequestContext.get()](/api/class-apirequestcontext.mdx#api-request-context-get) and similar methods are stored in the memory, so that you can later call [apiResponse.body()](/api/class-apiresponse.mdx#api-response-body).This method discards all its resources, calling any method on disposed [APIRequestContext] will throw an exception.

**使用方式**

```js
await apiRequestContext.dispose();
await apiRequestContext.dispose(options);
```

**參數**
- `options` [Object] *(optional)*
  - `reason` [string] *(optional)* <font size="2">Added in: v1.45</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-dispose-option-reason"/><a href="#api-request-context-dispose-option-reason" class="list-anchor">#</a>

### fetch {#api-request-context-fetch}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.fetch</x-search>

Sends HTTP(S) request and returns its response. The method will populate request cookies from the context and update context cookies from the response. The method will automatically follow redirects.

**使用方式**

JSON objects can be passed directly to the request:

```js
await request.fetch('https://example.com/api/createBook', {
  method: 'post',
  data: {
    title: 'Book Title',
    author: 'John Doe',
  }
});
```

The common way to send file(s) in the body of a request is to upload them as form fields with `multipart/form-data` encoding, by specifiying the `multipart` parameter:

```js
const form = new FormData();
form.set('name', 'John');
form.append('name', 'Doe');
// Send two file fields with the same name.
form.append('file', new File(['console.log(2024);'], 'f1.js', { type: 'text/javascript' }));
form.append('file', new File(['hello'], 'f2.txt', { type: 'text/plain' }));
await request.fetch('https://example.com/api/uploadForm', {
  multipart: form
});
```

### get {#api-request-context-get}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.get</x-search>

傳送 HTTP(S) [GET](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/GET) 請求並回傳其回應。此方法會從情境填入請求 cookie，並從回應更新情境 cookie。此方法會自動追蹤重新導向。

**使用方式**

Request parameters can be configured with `params` option, they will be serialized into the URL search parameters:

```js
// Passing params as object
await request.get('https://example.com/api/getText', {
  params: {
    'isbn': '1234',
    'page': 23,
  }
});

// Passing params as URLSearchParams
const searchParams = new URLSearchParams();
searchParams.set('isbn', '1234');
searchParams.append('page', 23);
searchParams.append('page', 24);
await request.get('https://example.com/api/getText', { params: searchParams });

// Passing params as string
const queryString = 'isbn=1234&page=23&page=24';
await request.get('https://example.com/api/getText', { params: queryString });
```

**使用方式**

JSON objects can be passed directly to the request:

```js
await request.post('https://example.com/api/createBook', {
  data: {
    title: 'Book Title',
    author: 'John Doe',
  }
});
```

To send form data to the server use `form` option. Its value will be encoded into the request body with `application/x-www-form-urlencoded` encoding (see below how to use `multipart/form-data` form encoding to send files):

```js
await request.post('https://example.com/api/findBook', {
  form: {
    title: 'Book Title',
    author: 'John Doe',
  }
});
```

The common way to send file(s) in the body of a request is to upload them as form fields with `multipart/form-data` encoding. Use [FormData] to construct request body and pass it to the request as `multipart` parameter:

```js
const form = new FormData();
form.set('name', 'John');
form.append('name', 'Doe');
// Send two file fields with the same name.
form.append('file', new File(['console.log(2024);'], 'f1.js', { type: 'text/javascript' }));
form.append('file', new File(['hello'], 'f2.txt', { type: 'text/plain' }));
await request.post('https://example.com/api/uploadForm', {
  multipart: form
});
```

### storageState {#api-request-context-storage-state}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.storageState</x-search>

Returns storage state for this request context, contains current cookies and local storage snapshot if it was passed to the constructor.

**使用方式**

```js
await apiRequestContext.storageState();
await apiRequestContext.storageState(options);
```

**參數**
- `options` [Object] *(optional)*
  - `indexedDB` [boolean] *(optional)* <font size="2">Added in: v1.51</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-option-indexed-db"/><a href="#api-request-context-storage-state-option-indexed-db" class="list-anchor">#</a>
    
    Set to `true` to include IndexedDB in the storage state snapshot.
  - `path` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-option-path"/><a href="#api-request-context-storage-state-option-path" class="list-anchor">#</a>
    
    儲存儲存狀態的檔案路徑。 If [path](/api/class-apirequestcontext.mdx#api-request-context-storage-state-option-path) is a relative path, then it is resolved relative to current working directory. If no path is provided, storage state is still returned, but won't be saved to the disk.

**回傳值**
- [Promise]&lt;[Object]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-return"/><a href="#api-request-context-storage-state-return" class="list-anchor">#</a>
  - `cookies` [Array]&lt;[Object]&gt;
    - `name` [string]
      
      
    - `value` [string]
      
      
    - `domain` [string]
      
      
    - `path` [string]
      
      
    - `expires` [number]
      
      Unix 時間，以秒為單位。
    - `httpOnly` [boolean]
      
      
    - `secure` [boolean]
      
      
    - `sameSite` "Strict" | "Lax" | "None"
      
      
    
  - `origins` [Array]&lt;[Object]&gt;
    - `origin` [string]
      
      
    - `localStorage` [Array]&lt;[Object]&gt;
      - `name` [string]
        
        
      - `value` [string]
