## 簡介

Playwright 具備為您產生測試的能力，當您在瀏覽器中執行操作時，它會自動錄製測試，這是快速開始測試的絕佳方式。Playwright 會檢視您的頁面並找出最佳的定位器，優先使用[角色、文字與測試 ID 定位器](./locators.mdx)。如果產生器發現多個元素符合定位器，它會改善定位器使其更穩固，以唯一識別目標元素。

## 使用 Playwright Inspector 產生測試

執行 `codegen` 指令時，將會開啟兩個視窗：一個是瀏覽器視窗（您可以在其中與想要測試的網站互動），另一個是 Playwright Inspector 視窗（您可以在其中錄製測試，然後將測試複製到您的編輯器中）。

### 執行 Codegen

使用 `codegen` 指令執行測試產生器，接著輸入您想要產生測試的網站 URL。URL 是可選的，您也可以不帶 URL 執行指令，然後直接在瀏覽器視窗中輸入 URL。

```bash
playwright codegen demo.playwright.dev/todomvc
```

### 錄製測試

執行 `codegen` 指令並在瀏覽器視窗中執行操作。Playwright 會為使用者互動產生程式碼，您可以在 Playwright Inspector 視窗中看到這些程式碼。錄製完測試後，停止錄製並按下 **copy** 按鈕，將產生的測試複製到您的編輯器中。

使用測試產生器，您可以錄製：
* 動作，例如點擊或填寫，只需與頁面互動即可
* 斷言，點擊工具列中的其中一個圖示，然後點擊頁面上的元素來進行斷言。您可以選擇：
  * `'assert visibility'` 斷言元素是可見的
  * `'assert text'` 斷言元素包含特定文字
  * `'assert value'` 斷言元素具有特定值

###### 
![recording a test](https://github.com/microsoft/playwright/assets/13063165/9751b609-6e4c-486b-a961-f86f177b1d58)

######
當您完成與頁面的互動後，按下 **record** 按鈕停止錄製，並使用 **copy** 按鈕將產生的程式碼複製到您的編輯器中。

使用 **clear** 按鈕清除程式碼，以便重新開始錄製。完成後，關閉 Playwright Inspector 視窗或停止終端指令。

### 產生定位器

您可以使用測試產生器產生[定位器](/locators.mdx)。
* 按下 `'Record'` 按鈕停止錄製，`'Pick Locator'` 按鈕就會出現。
* 點擊 `'Pick Locator'` 按鈕，然後將滑鼠移到瀏覽器視窗中的元素上方，您會看到每個元素下方都高亮顯示了定位器。
* 要選擇定位器，點擊您想要定位的元素，該定位器的程式碼就會出現在 Pick Locator 按鈕旁邊的欄位中。
* 然後您可以在此欄位中編輯定位器來微調它，或使用複製按鈕將其複製並貼上到您的程式碼中。

![picking a locator](https://github.com/microsoft/playwright/assets/13063165/95d11f48-96a4-46b9-9c2a-63c3aa4fdce7)

## 模擬

您可以使用測試產生器模擬產生測試，以為特定檢視區、裝置、顏色方案產生測試，也可以模擬地理位置、語言或時區。測試產生器也可以在保持驗證狀態的同時產生測試。

### 模擬檢視區大小

Playwright 開啟的瀏覽器視窗會將其檢視區設定為特定的寬度和高度，並且不會自適應，因為測試需要在相同條件下執行。使用 `--viewport` 選項可以產生不同檢視區大小的測試。

```bash
playwright codegen --viewport-size="800,600" playwright.dev
```

### 模擬裝置

使用 `--device` 選項模擬行動裝置時錄製腳本和測試，該選項會設定檢視區大小和使用者代理等。

```bash
playwright codegen --device="iPhone 13" playwright.dev
```

###### 
<img width="1300" alt="Codegen generating code for tests for playwright.dev website emulated for iPhone 13 python" src="https://user-images.githubusercontent.com/13063165/220922790-5c5a4d1a-e27d-4c9b-90ac-13cf9c925706.png" />

### 模擬顏色方案

使用 `--color-scheme` 選項模擬顏色方案時錄製腳本和測試。

```bash
playwright codegen --color-scheme=dark playwright.dev
```

###### 
<img width="1394" alt="Codegen generating code for tests for playwright.dev website in dark mode python" src="https://user-images.githubusercontent.com/13063165/220930714-737647fd-ae99-4dd3-b7a4-4f3eb4fe712d.png" />

### 模擬地理位置、語言和時區

使用 `--timezone`、`--geolocation` 和 `--lang` 選項模擬時區、語言和位置時錄製腳本和測試。頁面開啟後：
1. 接受 cookies
1. 在右上角點擊 locate me 按鈕查看地理位置作用。

```bash
playwright codegen --timezone="Europe/Rome" --geolocation="41.890221,12.492348" --lang="it-IT" bing.com/maps
```

###### 
<img width="1394" alt="Codegen generating code for tests for bing maps showing timezone, geolocation as Rome, Italy and in Italian language python" src="https://user-images.githubusercontent.com/13063165/220932413-f2943956-dd38-4560-94b9-51968076210d.png" />

### 保持驗證狀態

使用 `--save-storage` 執行 `codegen` 可在會話結束時儲存 [cookies](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)、[localStorage](https://developer.mozilla.org/en-US/docs/Web/API/Window/localStorage) 和 [IndexedDB](https://developer.mozilla.org/en-US/docs/Web/API/IndexedDB_API) 資料。這對於單獨錄製驗證步驟並在之後錄製更多測試時重複使用很有用。

```bash
playwright codegen github.com/microsoft/playwright --save-storage=auth.json
```

###### 
<img width="1394" alt="github page before logging in python" src="https://user-images.githubusercontent.com/13063165/220929429-8756ec49-fbf2-46e0-8f41-d25f5f5a6623.png" />

#### 登入

執行驗證並關閉瀏覽器後，`auth.json` 將包含儲存狀態，您可以在測試中重複使用。

<img width="1394" alt="login to GitHub screen" src="https://user-images.githubusercontent.com/13063165/220561688-04b2b984-4ba6-4446-8b0a-8058876e2a02.png" />

請確保您只在本機使用 `auth.json`，因為它包含敏感資訊。將其加入您的 `.gitignore` 或在完成產生測試後將其刪除。

#### 載入驗證狀態

使用 `--load-storage` 執行，來使用先前從 `auth.json` 載入的儲存。這樣，所有 [cookies](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)、[localStorage](https://developer.mozilla.org/en-US/docs/Web/API/Window/localStorage) 和 [IndexedDB](https://developer.mozilla.org/en-US/docs/Web/API/IndexedDB_API) 資料都會被還原，將大多數 Web 應用程式帶到驗證狀態，而無需再次登入。這表示您可以從登入狀態繼續產生測試。

```bash
playwright codegen --load-storage=auth.json github.com/microsoft/playwright
```

###### 
<img width="1394" alt="github signed in showing use of load storage python" src="https://user-images.githubusercontent.com/13063165/220928211-ca1d4dc9-9966-414e-ab23-a3ef1d2d5ed9.png" />

#### 使用現有的 userDataDir

使用 `--user-data-dir` 執行 `codegen`，為瀏覽器會話設定固定的[使用者資料目錄](https://playwright.dev/docs/api/class-browsertype#browser-type-launch-persistent-context-option-user-data-dir)。如果您建立自訂瀏覽器使用者資料目錄，codegen 將使用此現有瀏覽器設定檔，並能存取該設定檔中存在的任何驗證狀態。

:::warning

[從 Chrome 136 開始，預設使用者資料目錄無法透過自動化工具存取](https://developer.chrome.com/blog/remote-debugging-port)，例如 Playwright。您必須建立單獨的使用者資料目錄供測試使用。
:::

```bash
playwright codegen --user-data-dir=/path/to/your/browser/data/ github.com/microsoft/playwright
```

## 使用自訂設定錄製

如果您想在一些非標準設定中使用 codegen（例如，使用 [browser_context.route()](/api/class-browsercontext.mdx#browser-context-route)），可以呼叫 [page.pause()](/api/class-page.mdx#page-pause)，這會開啟一個帶有 codegen 控制項的單獨視窗。

<Tabs

  defaultValue="sync"

<TabItem value="sync">

```py
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    # 確保以有頭模式執行。
    browser = p.chromium.launch(headless=False)

    # 按您的喜好設定情境。
    context = browser.new_context() # 傳遞任何選項
    context.route('**/*', lambda route: route.continue_())

    # 暫停頁面，並手動開始錄製。
    page = context.new_page()
    page.pause()
```

</TabItem>
<TabItem value="async">

```py
import asyncio
from playwright.async_api import async_playwright

async def main():
    async with async_playwright() as p:
        # 確保以有頭模式執行。
        browser = await p.chromium.launch(headless=False)

        # 按您的喜好設定情境。
        context = await browser.new_context() # 傳遞任何選項
        await context.route('**/*', lambda route: route.continue_())

        # 暫停頁面，並手動開始錄製。
        page = await context.new_page()
        await page.pause()

asyncio.run(main())
```
