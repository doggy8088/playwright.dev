## 簡介

您可以執行單一測試、一組測試或所有測試。可使用 `--browser` 旗標在一個瀏覽器或多個瀏覽器上執行測試。測試預設以無頭模式執行，意即在執行測試時不會開啟瀏覽器視窗，結果顯示在終端機中。您可以使用 `--headed` CLI 參數在有頭模式下執行測試。

**您將學習**
- [如何從命令列執行測試](/running-tests.mdx#command-line)
- [如何偵錯測試](/running-tests.mdx#debugging-tests)

## 執行測試

### 命令列

要執行測試，請使用 `pytest` 指令。這會預設在 Chromium 瀏覽器上執行測試。測試預設以無頭模式執行，意即在執行測試時不會開啟瀏覽器視窗，結果顯示在終端機中。

```bash
pytest
```

### 在有頭模式下執行測試

要在有頭模式下執行測試，請使用 `--headed` 旗標。這會在執行測試時開啟瀏覽器視窗，完成後瀏覽器視窗會關閉。

```bash
pytest --headed
```

### 在不同瀏覽器上執行測試

要指定您想要執行測試的瀏覽器，請使用 `--browser` 旗標後接瀏覽器名稱。

```bash
pytest --browser webkit
```

要指定多個瀏覽器來執行測試，請多次使用 `--browser` 旗標，每次後接一個瀏覽器名稱。

```bash
pytest --browser webkit --browser firefox
```

### 執行特定測試

要執行單個測試文件，請傳入您想要執行的測試文件名稱。

```bash
pytest test_login.py
```

要執行一組測試文件，請傳入您想要執行的測試文件名稱。

```bash
pytest tests/test_todo_page.py tests/test_landing_page.py
```

要執行特定測試，請傳入您想要執行的測試函式名稱。

```bash
pytest -k test_add_a_todo_item
```

### 平行執行測試

要平行執行測試，請使用 `--numprocesses` 旗標後接您想要執行測試的處理程序數量。我們建議使用邏輯 CPU 核心數量的一半。

```bash
pytest --numprocesses 2
```

（這假設已安裝 `pytest-xdist`。更多資訊請參閱[這裡](./test-runners.mdx#parallelism-running-multiple-tests-at-once)。）

更多資訊請參閱 [Playwright Pytest 使用方法](./test-runners.mdx)或 Pytest 文件的[一般 CLI 使用方法](https://docs.pytest.org/en/stable/usage.html)。

## 偵錯測試

由於 Playwright 在 Python 中執行，您可以使用自己選擇的偵錯器進行偵錯，例如 Visual Studio Code 中的 [Python 擴充功能](https://code.visualstudio.com/docs/python/python-tutorial)。Playwright 附帶 Playwright Inspector，讓您能夠逐步執行 Playwright API 呼叫、查看偵錯記錄檔並探索[定位器](./locators.mdx)。

要偵錯所有測試，請執行以下指令。


</TabItem>
</Tabs>

此指令會開啟瀏覽器視窗以及 Playwright Inspector。您可以使用檢查器頂部的逐步執行按鈕來逐步執行測試。或按播放按鈕從頭到尾執行測試。測試完成後，瀏覽器視窗會關閉。

在偵錯時，您可以使用選擇定位器按鈕選取頁面上的元素，並查看 Playwright 用來找到該元素的定位器。您也可以編輯定位器並在瀏覽器視窗中看到即時反白顯示。使用複製定位器按鈕將定位器複製到剪貼簿，然後貼上到測試中。

![Playwright Inspector](https://github.com/microsoft/playwright/assets/13063165/c94c89c8-f945-460c-a653-7809c6ca50ee)

查看我們的[偵錯指南](./debug.mdx)以了解更多關於 [Playwright Inspector](./debug.mdx#playwright-inspector) 以及使用[瀏覽器開發者工具](./debug.mdx#browser-developer-tools)進行偵錯的資訊。
