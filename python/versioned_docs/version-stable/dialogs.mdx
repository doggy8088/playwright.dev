## 簡介

Playwright 可以與網頁對話框互動，例如 [`alert`](https://developer.mozilla.org/en-US/docs/Web/API/Window/alert)、[`confirm`](https://developer.mozilla.org/en-US/docs/Web/API/Window/confirm)、[`prompt`](https://developer.mozilla.org/en-US/docs/Web/API/Window/prompt) 以及 [`beforeunload`](https://developer.mozilla.org/en-US/docs/Web/API/Window/beforeunload_event) 確認。對於列印對話框，請參閱 [Print](#print-dialogs)。

## alert()、confirm()、prompt() 對話框

預設情況下，對話框會被 Playwright 自動關閉，所以您不需要處理它們。但是，您可以在觸發對話框的操作之前註冊對話框處理程序，以 [dialog.accept()](/api/class-dialog.mdx#dialog-accept) 或 [dialog.dismiss()](/api/class-dialog.mdx#dialog-dismiss) 它。

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.on("dialog", lambda dialog: dialog.accept())
page.get_by_role("button").click()
```

</TabItem>
<TabItem value="async">

```py
page.on("dialog", lambda dialog: dialog.accept())
await page.get_by_role("button".click())
```

</TabItem>
</Tabs>

:::note

[page.on("dialog")](/api/class-page.mdx#page-event-dialog) 監聽器 **必須處理** 對話框。否則您的操作將會停滯，無論是 [locator.click()](/api/class-locator.mdx#locator-click) 還是其他操作。這是因為網頁中的對話框是模態的，因此會阻止進一步的頁面執行，直到它們被處理。
:::

因此，以下程式碼片段永遠不會解析：

:::warning

錯誤！
:::

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.on("dialog", lambda dialog: print(dialog.message))
page.get_by_role("button").click() # 將會停滯在此
```

</TabItem>
<TabItem value="async">

```py
page.on("dialog", lambda dialog: print(dialog.message))
await page.get_by_role("button").click() # 將會停滯在此
```

</TabItem>
</Tabs>

:::note

如果沒有 [page.on("dialog")](/api/class-page.mdx#page-event-dialog) 的監聽器，所有對話框都會自動關閉。
:::

## beforeunload 對話框

當 [page.close()](/api/class-page.mdx#page-close) 以真值 [run_before_unload](/api/class-page.mdx#page-close-option-run-before-unload) 調用時，頁面會運行其卸載處理程序。這是 [page.close()](/api/class-page.mdx#page-close) 不等待頁面實際關閉的唯一情況，因為頁面可能最終保持打開狀態。

您可以註冊對話框處理程序來自己處理 `beforeunload` 對話框：

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
def handle_dialog(dialog):
    assert dialog.type == 'beforeunload'
    dialog.dismiss()

page.on('dialog', lambda: handle_dialog)
page.close(run_before_unload=True)
```

</TabItem>
<TabItem value="async">

```py
async def handle_dialog(dialog):
    assert dialog.type == 'beforeunload'
    await dialog.dismiss()

page.on('dialog', lambda: handle_dialog)
await page.close(run_before_unload=True)
```

</TabItem>
</Tabs>

## 列印對話框

為了斷言通過 [`window.print`](https://developer.mozilla.org/en-US/docs/Web/API/Window/print) 觸發的列印對話框，您可以使用以下程式碼片段：

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.goto("<url>")

page.evaluate("(() => {window.waitForPrintDialog = new Promise(f => window.print = f);})()")
page.get_by_text("Print it!").click()

page.wait_for_function("window.waitForPrintDialog")
```

</TabItem>
<TabItem value="async">

```py
await page.goto("<url>")

await page.evaluate("(() => {window.waitForPrintDialog = new Promise(f => window.print = f);})()")
await page.get_by_text("Print it!").click()

await page.wait_for_function("window.waitForPrintDialog")
```

</TabItem>
</Tabs>

這將在按鈕點擊後等待列印對話框打開。確保在點擊按鈕之前 / 頁面載入後評估腳本。
