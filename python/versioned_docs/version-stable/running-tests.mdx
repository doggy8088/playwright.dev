## 簡介

您可以執行單一測試、一組測試或所有測試。可以使用 `--browser` 標誌在一個瀏覽器或多個瀏覽器上執行測試。預設情況下，測試以無頭方式執行，這意味著在執行測試時不會打開瀏覽器視窗，結果將在終端機中顯示。如果您願意，可以使用 `--headed` CLI 參數在有頭模式下執行測試。

**你將學到**

- [如何從命令列執行測試](/running-tests.mdx#命令列)
- [如何調試測試](/running-tests.mdx#除錯測試)

## 執行測試

### 命令列

要執行你的測試，使用 `pytest` 命令。這將預設在 Chromium 瀏覽器上執行你的測試。測試預設在無頭模式下執行，這意味著在執行測試時不會打開瀏覽器視窗，結果將在終端機中顯示。

```bash
pytest
```

### 在有頭模式下執行測試

要在 headed 模式下執行你的測試，使用 `--headed` 標誌。這將在執行測試時打開一個瀏覽器視窗，測試完成後瀏覽器視窗將關閉。

```bash
pytest --headed
```

### 在不同瀏覽器上執行測試

要指定您想要在哪個瀏覽器上執行測試，請使用 `--browser` 標誌，後接瀏覽器的名稱。

```bash
pytest --browser webkit
```

要指定多個瀏覽器來執行你的測試，請多次使用 `--browser` 標誌，後跟每個瀏覽器的名稱。

```bash
pytest --browser webkit --browser firefox
```

### 執行特定測試

要 執行 單個 測試 文件, 請 傳入 您 想要 執行 的 測試 文件 名稱。

```bash
pytest test_login.py
```

要執行一組測試文件，請傳入您想要執行的測試文件名稱。

```bash
pytest tests/test_todo_page.py tests/test_landing_page.py
```

要執行特定測試，請傳入您想要執行的測試函式名稱。

```bash
pytest -k test_add_a_todo_item
```

### 平行執行測試

要平行執行測試，使用 `--numprocesses` 標誌，後接您想要執行測試的進程數。我們建議使用邏輯 CPU 核心的一半。

```bash
pytest --numprocesses 2
```

(這假設已安裝 `pytest-xdist`。更多資訊請參見 [此處](./test-runners.mdx#parallelism-running-multiple-tests-at-once) 。)

更多資訊，請參見 [Playwright Pytest 使用](./test-runners.mdx) 或 Pytest 文件中的 [一般 CLI 使用](https://docs.pytest.org/en/stable/usage.html)。

## 除錯測試

由於 Playwright 在 Python 中執行，您可以使用您選擇的偵錯工具來偵錯，例如，在 Visual Studio Code 中使用 [Python extension](https://code.visualstudio.com/docs/python/python-tutorial)。Playwright 附帶 Playwright Inspector，允許您逐步執行 Playwright API 呼叫，查看其偵錯日誌並探索 [locators](./locators.mdx)。

要除錯所有測試，請執行以下命令。

```batch
pytest -s
```

</TabItem>
</Tabs>

要調試一個測試文件，請執行命令並附上您要調試的測試文件名稱。

此命令將開啟一個瀏覽器視窗以及 Playwright Inspector。你可以使用檢查器頂部的步進按鈕來逐步執行你的測試。或者按下播放按鈕從頭到尾執行你的測試。一旦測試完成，瀏覽器視窗將關閉。

在偵錯時，你可以使用 Pick Locator 按鈕來選擇頁面上的元素，並查看 Playwright 將用來找到該元素的定位器。你也可以編輯定位器，並在瀏覽器視窗上即時看到它的高亮顯示。使用 Copy Locator 按鈕將定位器複製到剪貼簿，然後將其貼到你的測試中。

![Playwright Inspector](https://github.com/microsoft/playwright/assets/13063165/c94c89c8-f945-460c-a653-7809c6ca50ee)

請查看我們的[調試指南](./debug.mdx)以了解更多關於[Playwright Inspector](./debug.mdx#playwright-inspector)以及使用[瀏覽器開發者工具](./debug.mdx#browser-developer-tools)進行調試的資訊。
