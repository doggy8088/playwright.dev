此 API 用於 Web API 測試。您可以使用它來觸發 API 端點、設定微服務、準備環境或為您的端對端測試準備服務。

每個 Playwright 瀏覽器情境都有與其關聯的 [APIRequestContext] 實例，該實例與瀏覽器情境共享 cookie 儲存，並可透過 [browser_context.request](/api/class-browsercontext.mdx#browser-context-request) 或 [page.request](/api/class-page.mdx#page-request) 存取。也可以透過呼叫 [api_request.new_context()](/api/class-apirequest.mdx#api-request-new-context) 手動建立新的 APIRequestContext 實例。

**Cookie 管理**

由 [browser_context.request](/api/class-browsercontext.mdx#browser-context-request) 和 [page.request](/api/class-page.mdx#page-request) 回傳的 [APIRequestContext] 與對應的 [BrowserContext] 共享 cookie 儲存。每個 API 請求都會使用來自瀏覽器情境的值填入 `Cookie` 標頭。如果 API 回應包含 `Set-Cookie` 標頭，它會自動更新 [BrowserContext] cookie，並且從頁面發出的請求會選取它們。這意味著如果您使用此 API 登入，您的端對端測試將會被登入，反之亦然。

如果您希望 API 請求不干擾瀏覽器 cookie，您應該透過呼叫 [api_request.new_context()](/api/class-apirequest.mdx#api-request-new-context) 建立新的 [APIRequestContext]。這樣的 `APIRequestContext` 物件將擁有自己的隔離 cookie 儲存。

```py
import os
from playwright.sync_api import sync_playwright

REPO = "test-repo-1"
USER = "github-username"
API_TOKEN = os.getenv("GITHUB_API_TOKEN")

with sync_playwright() as p:
    # This will launch a new browser, create a context and page. When making HTTP
    # requests with the internal APIRequestContext (e.g. `context.request` or `page.request`)
    # it will automatically set the cookies to the browser page and vice versa.
    browser = p.chromium.launch()
    context = browser.new_context(base_url="https://api.github.com")
    api_request_context = context.request
    page = context.new_page()

    # Alternatively you can create a APIRequestContext manually without having a browser context attached:
    # api_request_context = p.request.new_context(base_url="https://api.github.com")


    # Create a repository.
    response = api_request_context.post(
        "/user/repos",
        headers={
            "Accept": "application/vnd.github.v3+json",
            # Add GitHub personal access token.
            "Authorization": f"token {API_TOKEN}",
        },
        data={"name": REPO},
    )
    assert response.ok
    assert response.json()["name"] == REPO

    # Delete a repository.
    response = api_request_context.delete(
        f"/repos/{USER}/{REPO}",
        headers={
            "Accept": "application/vnd.github.v3+json",
            # Add GitHub personal access token.
            "Authorization": f"token {API_TOKEN}",
        },
    )
    assert response.ok
    assert await response.body() == '{"status": "ok"}'
```

### dispose {#api-request-context-dispose}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.dispose</x-search>

### fetch {#api-request-context-fetch}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.fetch</x-search>

Sends HTTP(S) request and returns its response. The method will populate request cookies from the context and update context cookies from the response. The method will automatically follow redirects.

**使用方式**

JSON objects can be passed directly to the request:

```python
data = {
    "title": "Book Title",
    "body": "John Doe",
}
api_request_context.fetch("https://example.com/api/createBook", method="post", data=data)
```

The common way to send file(s) in the body of a request is to upload them as form fields with `multipart/form-data` encoding, by specifiying the `multipart` parameter:

```python
api_request_context.fetch(
  "https://example.com/api/uploadScript",  method="post",
  multipart={
    "fileField": {
      "name": "f.js",
      "mimeType": "text/javascript",
      "buffer": b"console.log(2022);",
    },
  })
```

```python
api_request_context.post(
```
