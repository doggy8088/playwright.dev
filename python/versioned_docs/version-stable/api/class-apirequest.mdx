公開可用於 Web API 測試的 API。此類別用於建立 [APIRequestContext] 實例，而該實例又可用於傳送網頁請求。此類別的實例可透過 [playwright.request](/api/class-playwright.mdx#playwright-request) 取得。如需更多資訊，請參閱 [APIRequestContext]。


---

## 方法

### new_context {#api-request-new-context}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequest.new_context</x-search>

建立新的 [APIRequestContext] 實例。

**使用方式**

```python
api_request.new_context()
api_request.new_context(**kwargs)
```

**參數**
- `base_url` [str] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-base-url"/><a href="#api-request-new-context-option-base-url" class="list-anchor">#</a>
  
  像 [api_request_context.get()](/api/class-apirequestcontext.mdx#api-request-context-get) 這樣的方法會使用 [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 建構函式來建構對應的 URL，從而將基礎 URL 納入考量。範例：
  * baseURL: `http://localhost:3000` 且傳送請求至 `/bar.html` 會產生 `http://localhost:3000/bar.html`
  * baseURL: `http://localhost:3000/foo/` 且傳送請求至 `./bar.html` 會產生 `http://localhost:3000/foo/bar.html`
  * baseURL: `http://localhost:3000/foo`（沒有尾隨斜線）且導航至 `./bar.html` 會產生 `http://localhost:3000/bar.html`
- `client_certificates` [List]\[[Dict]\] *(optional)* <font size="2">Added in: 1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-client-certificates"/><a href="#api-request-new-context-option-client-certificates" class="list-anchor">#</a>
  - `origin` [str]
    
    憑證適用的確切來源。來源包含 `https` 協定、主機名稱和可選的連接埠。
  - `certPath` [Union]\[[str], [pathlib.Path]\] *(optional)*
    
    PEM 格式憑證檔案的路徑。
  - `cert` [bytes] *(optional)*
    
    PEM 格式憑證的直接值。
  - `keyPath` [Union]\[[str], [pathlib.Path]\] *(optional)*
    
    PEM 格式私密金鑰檔案的路徑。
  - `key` [bytes] *(optional)*
    
    PEM 格式私密金鑰的直接值。
  - `pfxPath` [Union]\[[str], [pathlib.Path]\] *(optional)*
    
    PFX 或 PKCS12 編碼私密金鑰和憑證鏈的檔案路徑。
  - `pfx` [bytes] *(optional)*
    
    PFX 或 PKCS12 編碼私密金鑰和憑證鏈的直接值。
  - `passphrase` [str] *(optional)*
    
    私密金鑰的密碼（PEM 或 PFX）。
  
  TLS 用戶端驗證允許伺服器要求用戶端憑證並進行驗證。
  
  **詳細資訊**
  
  要使用的用戶端憑證陣列。每個憑證物件必須同時具有 `certPath` 和 `keyPath`、單一 `pfxPath`，或其對應的直接值等效項（`cert` 和 `key`，或 `pfx`）。如果憑證已加密，則應選擇性地提供 `passphrase` 屬性。應該提供 `origin` 屬性，以確切符合憑證適用的請求來源。
  
  只有在提供至少一個用戶端憑證時，用戶端憑證驗證才會啟用。如果您想要拒絕伺服器傳送的所有用戶端憑證，您需要提供一個 `origin` 與您計劃造訪的任何網域都不符合的用戶端憑證。
  
  :::note
  
  在 macOS 上使用 WebKit 時，存取 `localhost` 將不會選取用戶端憑證。您可以透過將 `localhost` 替換為 `local.playwright` 來使其正常運作。
  :::
  
- `extra_http_headers` [Dict]\[[str], [str]\] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-extra-http-headers"/><a href="#api-request-new-context-option-extra-http-headers" class="list-anchor">#</a>
  
  包含與每個請求一起傳送的額外 HTTP 標頭的物件。預設為無。
- `fail_on_status_code` [bool] *(optional)* <font size="2">Added in: v1.51</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-fail-on-status-code"/><a href="#api-request-new-context-option-fail-on-status-code" class="list-anchor">#</a>
  
  是否對 2xx 和 3xx 以外的回應代碼拋出例外。預設情況下，會為所有狀態碼回傳回應物件。
- `http_credentials` [Dict] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-http-credentials"/><a href="#api-request-new-context-option-http-credentials" class="list-anchor">#</a>
  - `username` [str]
    
    
  - `password` [str]
    
    
  - `origin` [str] *(optional)*
    
    Restrain sending http credentials on specific origin (scheme://host:port).
  - `send` "unauthorized" | "always" *(optional)*
    
    This option only applies to the requests sent from corresponding [APIRequestContext] and does not affect requests sent from the browser. `'always'` - `Authorization` header with basic authentication credentials will be sent with the each API request. `'unauthorized` - the credentials are only sent when 401 (Unauthorized) response with `WWW-Authenticate` header is received. Defaults to `'unauthorized'`.
  
  Credentials for [HTTP authentication](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication). If no origin is specified, the username and password are sent to any servers upon unauthorized responses.
- `ignore_https_errors` [bool] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-ignore-https-errors"/><a href="#api-request-new-context-option-ignore-https-errors" class="list-anchor">#</a>
  
  Whether to ignore HTTPS errors when sending network requests. Defaults to `false`.
- `max_redirects` [int] *(optional)* <font size="2">Added in: v1.52</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-max-redirects"/><a href="#api-request-new-context-option-max-redirects" class="list-anchor">#</a>
  
  Maximum number of request redirects that will be followed automatically. An error will be thrown if the number is exceeded. Defaults to `20`. Pass `0` to not follow redirects. This can be overwritten for each request individually.
- `proxy` [Dict] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-proxy"/><a href="#api-request-new-context-option-proxy" class="list-anchor">#</a>
  - `server` [str]
    
    用於所有請求的代理伺服器。支援 HTTP 和 SOCKS 代理伺服器，例如 `http://myproxy.com:3128` or `socks5://myproxy.com:3128`。簡短形式 `myproxy.com:3128` 被視為 HTTP 代理伺服器。
  - `bypass` [str] *(optional)*
    
    Optional comma-separated domains to bypass proxy, for example `".com, chromium.org, .domain.com"`.
  - `username` [str] *(optional)*
    
    如果 HTTP 代理伺服器需要驗證，則為可選的使用者名稱。
  - `password` [str] *(optional)*
    
    如果 HTTP 代理伺服器需要驗證，則為可選的密碼。
  
  網路代理伺服器設定。
- `storage_state` [Union]\[[str], [pathlib.Path]\] | [Dict] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-storage-state"/><a href="#api-request-new-context-option-storage-state" class="list-anchor">#</a>
  - `cookies` [List]\[[Dict]\]
    - `name` [str]
      
      
    - `value` [str]
      
      
    - `domain` [str]
      
      
    - `path` [str]
      
      
    - `expires` [float]
      
      Unix 時間，以秒為單位。
    - `httpOnly` [bool]
      
      
    - `secure` [bool]
      
      
    - `sameSite` "Strict" | "Lax" | "None"
      
      
    
  - `origins` [List]\[[Dict]\]
    - `origin` [str]
      
      
    - `localStorage` [List]\[[Dict]\]
      - `name` [str]
        
        
      - `value` [str]
        
        
      
    
  Populates context with given storage state. This option can be used to initialize context with logged-in information obtained via [browser_context.storage_state()](/api/class-browsercontext.mdx#browser-context-storage-state) or [api_request_context.storage_state()](/api/class-apirequestcontext.mdx#api-request-context-storage-state). Either a path to the file with saved storage, or the value returned by one of [browser_context.storage_state()](/api/class-browsercontext.mdx#browser-context-storage-state) or [api_request_context.storage_state()](/api/class-apirequestcontext.mdx#api-request-context-storage-state) methods.
- `timeout` [float] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-timeout"/><a href="#api-request-new-context-option-timeout" class="list-anchor">#</a>
  
  等待回應的最大時間（以毫秒為單位）。預設為 `30000`（30 秒）。傳遞 `0` 以停用逾時。
- `user_agent` [str] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-user-agent"/><a href="#api-request-new-context-option-user-agent" class="list-anchor">#</a>
  
  在此情境中使用的特定使用者代理。
