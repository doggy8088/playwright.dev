[APIRequestContext] returned by [BrowserContext.request()](https://api/class-browsercontext.mdx#browser-context-request) and [Page.request()](https://api/class-page.mdx#page-request) shares cookie storage with the corresponding [BrowserContext]。Each API request will have `Cookie` header populated with the values from the browser context。如果 API response contains `Set-Cookie` header it will automatically update [BrowserContext] cookies and requests made from the page will pick them up。這意味著如果你使用這個 API 登入，你的 e2e 測試將會登入，反之亦然。
