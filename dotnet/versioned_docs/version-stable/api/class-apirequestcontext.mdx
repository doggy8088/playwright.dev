此 API 用於 Web API 測試。您可以使用它來觸發 API 端點、組態設定微服務、準備環境或服務以進行端對端測試。

每個 Playwright 瀏覽器情境都有一個關聯的 [APIRequestContext] 實例，它與瀏覽器情境共享 cookie 儲存，並可透過 [BrowserContext.APIRequest](/api/class-browsercontext.mdx#browser-context-request) 或 [Page.APIRequest](/api/class-page.mdx#page-request) 存取。也可以透過呼叫 [ApiRequest.NewContextAsync()](/api/class-apirequest.mdx#api-request-new-context) 手動建立新的 `APIRequestContext` 實例。

**Cookie 管理**

由 [BrowserContext.APIRequest](/api/class-browsercontext.mdx#browser-context-request) 和 [Page.APIRequest](/api/class-page.mdx#page-request) 傳回的 [APIRequestContext] 與對應的 [BrowserContext] 共享 cookie 儲存。每個 API 請求都會有 `Cookie` 標頭，填入來自瀏覽器情境的值。如果 API 回應包含 `Set-Cookie` 標頭，它會自動更新 [BrowserContext] cookies，並且從頁面發出的請求會取得它們。這意味著如果您使用此 API 登入，您的端對端測試將會登入，反之亦然。

如果您想要 API 請求不干擾瀏覽器 cookies，您應該透過呼叫 [ApiRequest.NewContextAsync()](/api/class-apirequest.mdx#api-request-new-context) 建立新的 [APIRequestContext]。這樣的 `APIRequestContext` 物件將擁有自己的隔離 cookie 儲存。

### DeleteAsync {#api-request-context-delete}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.DeleteAsync</x-search>

```csharp

```

**使用方式**

JSON 物件可以直接傳遞給請求：

```csharp

  { "title", "Book Title" },
  { "body", "John Doe" }
};
await Request.FetchAsync("https://example.com/api/createBook", new() { Method = "post", DataObject = data });
```

在請求主體中傳送檔案的常見方式是透過指定 `multipart` 參數，將它們作為 `multipart/form-data` 編碼的表單欄位上傳：

```csharp
var file = new FilePayload()
{
    Name = "f.js",
    MimeType = "text/javascript",
    Buffer = System.Text.Encoding.UTF8.GetBytes("console.log(2022);")
};
var multipart = Context.APIRequest.CreateFormData();
multipart.Set("fileField", file);
await Request.FetchAsync("https://example.com/api/uploadScript", new() { Method = "post", Multipart = multipart });
```

};

formData.Set("body", "John Doe");

### PutAsync {#api-request-context-put}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.PutAsync</x-search>

傳送 HTTP(S) [PUT](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/PUT) 請求並傳回其回應。此方法會從情境填入請求 cookies，並從回應更新情境 cookies。此方法會自動跟隨重新導向。

**使用方式**

```csharp
await ApiRequestContext.PutAsync(url, options);
```

**參數**
- `url` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-url"/><a href="#api-request-context-put-option-url" class="list-anchor">#</a>
  
  目標 URL。
- `options` `ApiRequestContextPutOptions?` *(選用)*
  - `Data|DataByte|DataObject` [string]? | [byte]&#91;&#93;? | [object]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-data"/><a href="#api-request-context-put-option-data" class="list-anchor">#</a>
    
    允許設定請求的 post 資料。如果 data 參數是物件，它會被序列化為 json 字串，並且如果未明確設定，`content-type` 標頭會設定為 `application/json`。否則如果未明確設定，`content-type` 標頭會設定為 `application/octet-stream`。
  - `FailOnStatusCode` [bool]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-fail-on-status-code"/><a href="#api-request-context-put-option-fail-on-status-code" class="list-anchor">#</a>
    
    是否在回應碼不是 2xx 和 3xx 時拋出例外。預設情況下，所有狀態碼都會傳回回應物件。
  - `Form` [FormData]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-form"/><a href="#api-request-context-put-option-form" class="list-anchor">#</a>
    
    提供一個物件，該物件會使用 `application/x-www-form-urlencoded` 編碼序列化為 html 表單，並作為此請求主體傳送。如果指定此參數，除非明確提供，否則 `content-type` 標頭會設定為 `application/x-www-form-urlencoded`。
    
    [FormData] 實例可以透過 [ApiRequestContext.CreateFormData] 建立(/api/class-apirequestcontext.mdx#api-request-context-create-form-data).
  - `Headers` [IDictionary]?&lt;[string], [string]&gt; *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-headers"/><a href="#api-request-context-put-option-headers" class="list-anchor">#</a>
    
    允許設定 HTTP 標頭。這些標頭會套用到取得的請求以及任何由其起始的重新導向。
  - `IgnoreHTTPSErrors` [bool]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-ignore-https-errors"/><a href="#api-request-context-put-option-ignore-https-errors" class="list-anchor">#</a>
    
    傳送網路請求時是否忽略 HTTPS 錯誤。預設為 `false`。
  - `MaxRedirects` [int]? *(選用)* <font size="2">Added in: v1.26</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-max-redirects"/><a href="#api-request-context-put-option-max-redirects" class="list-anchor">#</a>
    
    將自動跟隨的請求重新導向最大數量。如果超出數量將拋出錯誤。預設為 `20`。傳遞 `0` 表示不跟隨重新導向。
  - `MaxRetries` [int]? *(選用)* <font size="2">Added in: v1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-max-retries"/><a href="#api-request-context-put-option-max-retries" class="list-anchor">#</a>
    
    網路錯誤應重試的最大次數。目前僅重試 `ECONNRESET` 錯誤。不會根據 HTTP 回應碼重試。如果超出限制將拋出錯誤。預設為 `0` - 不重試。
  - `Multipart` [FormData]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-multipart"/><a href="#api-request-context-put-option-multipart" class="list-anchor">#</a>
    
    提供一個物件，該物件會使用 `multipart/form-data` 編碼序列化為 html 表單，並作為此請求主體傳送。如果指定此參數，除非明確提供，否則 `content-type` 標頭會設定為 `multipart/form-data`。檔案值可以作為包含檔案名稱、mime 類型和其內容的類檔案物件傳遞。
    
    [FormData] 實例可以透過 [ApiRequestContext.CreateFormData] 建立(/api/class-apirequestcontext.mdx#api-request-context-create-form-data).
  - `Params` [IDictionary]?&lt;[string], [object]&gt; *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-params"/><a href="#api-request-context-put-option-params" class="list-anchor">#</a>
    
    要與 URL 一起傳送的查詢參數。
  - `ParamsString` [string]? *(選用)* <font size="2">Added in: v1.47</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-params-string"/><a href="#api-request-context-put-option-params-string" class="list-anchor">#</a>
    
    要與 URL 一起傳送的查詢參數。
  - `Timeout` [float]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-option-timeout"/><a href="#api-request-context-put-option-timeout" class="list-anchor">#</a>
    
    請求逾時時間，以毫秒為單位。預設為 `30000`（30 秒）。傳遞 `0` 表示停用逾時。

**傳回值**
- [APIResponse]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-put-return"/><a href="#api-request-context-put-return" class="list-anchor">#</a>

### StorageStateAsync {#api-request-context-storage-state}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequestContext.StorageStateAsync</x-search>

傳回此請求情境的儲存狀態，包含目前的 cookies 和本機儲存快照（如果已傳遞給建構函式）。

**使用方式**

```csharp
await ApiRequestContext.StorageStateAsync(options);
```

**參數**
- `options` `ApiRequestContextStorageStateOptions?` *(選用)*
  - `IndexedDB` [bool]? *(選用)* <font size="2">Added in: v1.51</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-option-indexed-db"/><a href="#api-request-context-storage-state-option-indexed-db" class="list-anchor">#</a>
    
    設定為 `true` 以在儲存狀態快照中包含 IndexedDB。
  - `Path` [string]? *(選用)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-option-path"/><a href="#api-request-context-storage-state-option-path" class="list-anchor">#</a>
    
    The file path to save the storage state to. If [Path](/api/class-apirequestcontext.mdx#api-request-context-storage-state-option-path) is a relative path, then it is resolved relative to current working directory. If no path is provided, storage state is still returned, but won't be saved to the disk.

**傳回值**
- [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-context-storage-state-return"/><a href="#api-request-context-storage-state-return" class="list-anchor">#</a>
