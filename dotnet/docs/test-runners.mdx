## 簡介

雖然 Playwright for .NET 並不綁定特定的測試執行器或測試框架，但根據我們的經驗，最簡單的入門方式是使用我們為 MSTest、NUnit、xUnit 或 xUnit v3 提供的基底類別。這些類別支援在多個瀏覽器引擎上執行測試、調整啟動/情境選項，並為每個測試提供 [Page]/[BrowserContext] 實例。

Playwright 和 Browser 實例將在測試之間重複使用以獲得更好的效能。我們建議在新的 BrowserContext 中執行每個測試案例，這樣瀏覽器狀態就會在測試之間被隔離。

## 平行執行測試

<Tabs groupId="test-runners" defaultValue="mstest" values={[ {label: 'MSTest', value: 'mstest'}, {label: 'NUnit', value: 'nunit'}, {label: 'xUnit', value: 'xunit'}, {label: 'xUnit v3', value: 'xunit-v3'}, ] }>

<TabItem value="nunit">

預設情況下，NUnit 會平行執行所有測試檔案，而在每個檔案內循序執行測試（`ParallelScope.Self`）。它會建立與主機系統核心數相同的工作程序。您可以使用 NUnit.NumberOfTestWorkers 參數來調整此行為。僅支援 `ParallelScope.Self`。

對於 CPU 密集型測試，我們建議使用系統核心數除以 2 的工作程序數。對於 IO 密集型測試，您可以使用與核心數相同的工作程序數。

```bash
dotnet test -- NUnit.NumberOfTestWorkers=5
```

</TabItem>

<TabItem value="mstest">

預設情況下，MSTest 會平行執行所有類別，而在每個類別內循序執行測試（`ExecutionScope.ClassLevel`）。它會建立與主機系統核心數相同的工作程序。您可以使用以下 CLI 參數或使用 `.runsettings` 檔案來調整此行為，請參見下文。不支援在方法層級平行執行測試（`ExecutionScope.MethodLevel`）。

```bash
dotnet test --settings:.runsettings -- MSTest.Parallelize.Workers=4
```

</TabItem>

<TabItem value="xunit">

預設情況下，xUnit 會平行執行所有類別，而在每個類別內循序執行測試。預設會建立與系統核心數相同的工作程序。您可以使用以下 CLI 參數或使用 `.runsettings` 檔案來調整此行為，請參見下文。

```bash
dotnet test -- xUnit.MaxParallelThreads=5
```

:::note
我們建議使用 xUnit 2.8+，它預設使用[`conservative` 平行算法](https://xunit.net/docs/running-tests-in-parallel.html#algorithms)。
:::

</TabItem>

<TabItem value="xunit-v3">

預設情況下，xUnit v3 會平行執行所有類別，而在每個類別內循序執行測試。預設會建立與系統核心數相同的工作程序。您可以使用以下 CLI 參數或使用 `.runsettings` 檔案來調整此行為，請參見下文。

```bash
dotnet test -- xUnit.MaxParallelThreads=5
```

:::note
xUnit v3 預設使用[`conservative` 平行算法](https://xunit.net/docs/running-tests-in-parallel.html#algorithms)。
:::

</TabItem>

</Tabs>

## 自訂 [BrowserContext] 選項

<Tabs groupId="test-runners" defaultValue="mstest" values={[ {label: 'MSTest', value: 'mstest'}, {label: 'NUnit', value: 'nunit'}, {label: 'xUnit', value: 'xunit'}, {label: 'xUnit v3', value: 'xunit-v3'}, ] }>

<TabItem value="nunit">

要自訂情境選項，您可以覆寫繼承自 `Microsoft.Playwright.MSTest.PageTest` 或 `Microsoft.Playwright.MSTest.ContextTest` 測試類別的 `ContextOptions` 方法。請參閱以下範例：

```csharp
using Microsoft.Playwright.NUnit;

namespace PlaywrightTests;

[Parallelizable(ParallelScope.Self)]
[TestFixture]
public class MyTest : PageTest
{
    [Test]
    public async Task TestWithCustomContextOptions()
    {
        // The following Page (and BrowserContext) instance has the custom colorScheme, viewport and baseURL set:
        await Page.GotoAsync("/login");
    }

    public override BrowserNewContextOptions ContextOptions()
    {
        return new BrowserNewContextOptions()
        {
            ColorScheme = ColorScheme.Light,
            ViewportSize = new()
            {
                Width = 1920,
                Height = 1080
            },
            BaseURL = "https://github.com",
        };
    }
}
```

</TabItem>

<TabItem value="mstest">

要自訂情境選項，您可以覆寫繼承自 `Microsoft.Playwright.MSTest.PageTest` 或 `Microsoft.Playwright.MSTest.ContextTest` 測試類別的 `ContextOptions` 方法。請參閱以下範例：

## 自訂 [Browser]/啟動選項

[Browser]/啟動選項可以使用執行設定檔案或直接透過 CLI 設定執行設定選項來覆寫。請參閱以下範例：

```xml
<?xml version="1.0" encoding="utf-8"?>
<RunSettings>
  <Playwright>
    <BrowserName>chromium</BrowserName>
    <LaunchOptions>
      <Headless>false</Headless>
      <Channel>msedge</Channel>
    </LaunchOptions>
  </Playwright>
</RunSettings>
```

```bash
dotnet test -- Playwright.BrowserName=chromium Playwright.LaunchOptions.Headless=false Playwright.LaunchOptions.Channel=msedge
```

## 使用詳細 API 記錄

當您透過 `DEBUG` 環境變數啟用[詳細 API 記錄](./debug.mdx#verbose-api-logs)時，您會在標準錯誤流中看到訊息。在 Visual Studio 中，這會是 `Output` 視窗的 `Tests` 窗格。它也會顯示在每個測試的 `Test Log` 中。

## 使用 .runsettings 檔案

在 Visual Studio 中執行測試時，您可以利用 `.runsettings` 檔案。以下顯示支援值的參考。

<Tabs groupId="test-runners" defaultValue="mstest" values={[ {label: 'MSTest', value: 'mstest'}, {label: 'NUnit', value: 'nunit'}, {label: 'xUnit', value: 'xunit'}, {label: 'xUnit v3', value: 'xunit-v3'}, ] }>

<TabItem value="nunit">

例如，要指定工作程序數量，您可以使用 `NUnit.NumberOfTestWorkers`，或要啟用 `DEBUG` 記錄，可使用 `RunConfiguration.EnvironmentVariables`。

```xml
<?xml version="1.0" encoding="utf-8"?>
<RunSettings>
  <!-- NUnit adapter -->  
  <NUnit>
    <NumberOfTestWorkers>24</NumberOfTestWorkers>
  </NUnit>
  <!-- General run configuration -->
  <RunConfiguration>
    <EnvironmentVariables>
      <!-- For debugging selectors, it's recommend to set the following environment variable -->
      <DEBUG>pw:api</DEBUG>
    </EnvironmentVariables>
  </RunConfiguration>
  <!-- Playwright -->  
  <Playwright>
    <BrowserName>chromium</BrowserName>
    <ExpectTimeout>5000</ExpectTimeout>
    <LaunchOptions>
      <Headless>false</Headless>
      <Channel>msedge</Channel>
    </LaunchOptions>
  </Playwright>
</RunSettings>
```

</TabItem>

<TabItem value="mstest">

例如，要指定工作程序數量，您可以使用 `MSTest.Parallelize.Workers`。您也可以使用 `RunConfiguration.EnvironmentVariables` 啟用 `DEBUG` 記錄。

```xml
<RunSettings>
  <!-- MSTest adapter -->  
  <MSTest>
    <Parallelize>
      <Workers>4</Workers>
      <Scope>ClassLevel</Scope>
    </Parallelize>
  </MSTest>
  <!-- General run configuration -->
  <RunConfiguration>
    <EnvironmentVariables>
      <!-- For debugging selectors, it's recommend to set the following environment variable -->
      <DEBUG>pw:api</DEBUG>
    </EnvironmentVariables>
  </RunConfiguration>
  <!-- Playwright -->  
  <Playwright>
    <BrowserName>chromium</BrowserName>
    <ExpectTimeout>5000</ExpectTimeout>
    <LaunchOptions>
      <Headless>false</Headless>
      <Channel>msedge</Channel>
    </LaunchOptions>
  </Playwright>
</RunSettings>
```

</TabItem>

<TabItem value="xunit">
