公開可用於 Web API 測試的 API。此類別用於建立 [APIRequestContext] 執行個體，該執行個體可用於傳送 Web 請求。可透過 [Playwright.APIRequest](/api/class-playwright.mdx#playwright-request) 取得此類別的執行個體。如需更多資訊，請參閱 [APIRequestContext]。


---

## 方法

### NewContextAsync {#api-request-new-context}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.16</font><x-search>apiRequest.NewContextAsync</x-search>

建立新的 [APIRequestContext] 執行個體。

**使用方式**

```csharp
await ApiRequest.NewContextAsync(options);
```

**參數**
- `options` `ApiRequestNewContextOptions?` *(optional)*
  - `BaseURL` [string]? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-base-url"/><a href="#api-request-new-context-option-base-url" class="list-anchor">#</a>
    
    像 [ApiRequestContext.GetAsync()](/api/class-apirequestcontext.mdx#api-request-context-get) 這樣的方法會透過使用 [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) 建構函式來建構對應的 URL，將基底 URL 納入考量。範例：
    * baseURL: `http://localhost:3000` 並傳送請求到 `/bar.html` 會產生 `http://localhost:3000/bar.html`
    * baseURL: `http://localhost:3000/foo/` 並傳送請求到 `./bar.html` 會產生 `http://localhost:3000/foo/bar.html`
    * baseURL: `http://localhost:3000/foo` (沒有結尾斜線) 並導航到 `./bar.html` 會產生 `http://localhost:3000/bar.html`
  - `ClientCertificates` [IEnumerable]?&lt;ClientCertificates&gt; *(optional)* <font size="2">Added in: 1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-client-certificates"/><a href="#api-request-new-context-option-client-certificates" class="list-anchor">#</a>
    - `Origin` [string]
      
      憑證有效的確切來源。來源包含 `https` 通訊協定、主機名稱及可選的連接埠。
    - `CertPath` [string]? *(optional)*
      
      PEM 格式憑證檔案的路徑。
    - `Cert` [byte]&#91;&#93;? *(optional)*
      
      PEM 格式憑證的直接值。
    - `KeyPath` [string]? *(optional)*
      
      PEM 格式私密金鑰檔案的路徑。
    - `Key` [byte]&#91;&#93;? *(optional)*
      
      PEM 格式私密金鑰的直接值。
    - `PfxPath` [string]? *(optional)*
      
      PFX 或 PKCS12 編碼私密金鑰和憑證鏈檔案的路徑。
    - `Pfx` [byte]&#91;&#93;? *(optional)*
      
      PFX 或 PKCS12 編碼私密金鑰和憑證鏈的直接值。
    - `Passphrase` [string]? *(optional)*
      
      私密金鑰的密碼 (PEM 或 PFX)。
    
    TLS 用戶端驗證允許伺服器請求用戶端憑證並驗證它。
    
    **詳細說明**
    
    要使用的用戶端憑證陣列。每個憑證物件必須同時具有 `certPath` 和 `keyPath`、單個 `pfxPath`，或其對應的直接值等效項（`cert` 和 `key`，或 `pfx`）。如果憑證已加密，可選擇提供 `passphrase` 屬性。應提供 `origin` 屬性以與憑證有效的請求來源完全匹配。
    
    用戶端憑證驗證只有在至少提供一個用戶端憑證時才會啟動。如果您想拒絕伺服器傳送的所有用戶端憑證，您需要提供一個 `origin` 與您計劃造訪的任何網域都不匹配的用戶端憑證。
    
    :::note
    
    在 macOS 上使用 WebKit 時，存取 `localhost` 不會取得用戶端憑證。您可以透過將 `localhost` 替換為 `local.playwright` 來使其工作。
    :::
    
  - `ExtraHTTPHeaders` [IDictionary]?&lt;[string], [string]&gt; *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-extra-http-headers"/><a href="#api-request-new-context-option-extra-http-headers" class="list-anchor">#</a>
    
    包含每個請求要傳送的額外 HTTP 標頭的物件。預設為無。
  - `FailOnStatusCode` [bool]? *(optional)* <font size="2">Added in: v1.51</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-fail-on-status-code"/><a href="#api-request-new-context-option-fail-on-status-code" class="list-anchor">#</a>
    
    是否對 2xx 和 3xx 以外的回應代碼拋出錯誤。預設情況下會為所有狀態代碼回傳回應物件。
  - `HttpCredentials` HttpCredentials? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-http-credentials"/><a href="#api-request-new-context-option-http-credentials" class="list-anchor">#</a>
    - `Username` [string]
      
      
    - `Password` [string]
      
      
    - `Origin` [string]? *(optional)*
      
      限制在特定來源 (scheme://host:port) 傳送 http 憑證。
    - `Send` `enum HttpCredentialsSend { Unauthorized, Always }?` *(optional)*
      
      此選項僅適用於從對應 [APIRequestContext] 傳送的請求，不影響從瀏覽器傳送的請求。`'always'` - 包含基本驗證憑證的 `Authorization` 標頭將隨每個 API 請求傳送。`'unauthorized'` - 憑證只有在收到帶有 `WWW-Authenticate` 標頭的 401 (Unauthorized) 回應時才會傳送。預設為 `'unauthorized'`。
    
    [HTTP 驗證](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication) 的憑證。如果未指定來源，使用者名稱和密碼會在未授權回應時傳送到任何伺服器。
  - `IgnoreHTTPSErrors` [bool]? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-ignore-https-errors"/><a href="#api-request-new-context-option-ignore-https-errors" class="list-anchor">#</a>
    
    傳送網路請求時是否忽略 HTTPS 錯誤。預設為 `false`。
  - `MaxRedirects` [int]? *(optional)* <font size="2">Added in: v1.52</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-max-redirects"/><a href="#api-request-new-context-option-max-redirects" class="list-anchor">#</a>
    
    自動跟隨的請求重新導向次數上限。如果超過此數目將拋出錯誤。預設為 `20`。傳遞 `0` 表示不跟隨重新導向。此設定可以針對每個請求個別覆寫。
  - `Proxy` Proxy? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-proxy"/><a href="#api-request-new-context-option-proxy" class="list-anchor">#</a>
    - `Server` [string]
      
      用於所有請求的代理。支援 HTTP 和 SOCKS 代理，例如 `http://myproxy.com:3128` 或 `socks5://myproxy.com:3128`。簡短形式 `myproxy.com:3128` 被視為 HTTP 代理。
    - `Bypass` [string]? *(optional)*
      
      可選的逗號分隔網域以繞過代理，例如 `".com, chromium.org, .domain.com"`。
    - `Username` [string]? *(optional)*
      
      如果 HTTP 代理需要驗證時使用的可選使用者名稱。
    - `Password` [string]? *(optional)*
      
      如果 HTTP 代理需要驗證時使用的可選密碼。
    
    網路代理設定。
  - `StorageState` [string]? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-storage-state"/><a href="#api-request-new-context-option-storage-state" class="list-anchor">#</a>
    
    使用給定的儲存狀態填入情境。此選項可用於透過 [BrowserContext.StorageStateAsync()](/api/class-browsercontext.mdx#browser-context-storage-state) 或 [ApiRequestContext.StorageStateAsync()](/api/class-apirequestcontext.mdx#api-request-context-storage-state) 取得的登入資訊來初始化情境。可以是儲存檔案的路徑，或是 [BrowserContext.StorageStateAsync()](/api/class-browsercontext.mdx#browser-context-storage-state) 或 [ApiRequestContext.StorageStateAsync()](/api/class-apirequestcontext.mdx#api-request-context-storage-state) 方法回傳的值。
  - `StorageStatePath` [string]? *(optional)* <font size="2">Added in: v1.18</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-storage-state-path"/><a href="#api-request-new-context-option-storage-state-path" class="list-anchor">#</a>
    
    使用給定的儲存狀態填入情境。此選項可用於透過 [BrowserContext.StorageStateAsync()](/api/class-browsercontext.mdx#browser-context-storage-state) 取得的登入資訊來初始化情境。儲存狀態檔案的路徑。
  - `Timeout` [float]? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-timeout"/><a href="#api-request-new-context-option-timeout" class="list-anchor">#</a>
    
    等待回應的最大時間（毫秒）。預設為 `30000`（30 秒）。傳遞 `0` 可停用逾時。
  - `UserAgent` [string]? *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="api-request-new-context-option-user-agent"/><a href="#api-request-new-context-option-user-agent" class="list-anchor">#</a>
    
    在此情境中使用的特定使用者代理程式。
