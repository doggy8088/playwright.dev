## 簡介

Playwright 測試可以在 CI 環境中執行。我們已為常見的 CI 提供者建立了範例設定。

在 CI 上執行測試的三個步驟：
1. **確保 CI 代理可以執行瀏覽器**：在 Linux 代理中使用[我們的 Docker 映像](./docker.mdx)，或使用 [CLI](./browsers#install-system-dependencies) 安裝您的相依性。
1. **安裝 Playwright**：
   
   ```bash
   dotnet build
   pwsh bin/Debug/netX/playwright.ps1 install --with-deps
   ```
   
1. **執行您的測試**：
   
   ```bash
   dotnet test
   ```
   
## CI 設定

[命令列工具](./browsers#install-system-dependencies)可用於在 CI 中安裝所有作業系統相依性。

### GitHub Actions

#### 在 push/pull_request 時

測試將在主要/主分支的推送或 pull request 時執行。[workflow](https://docs.github.com/en/actions/using-workflows/about-workflows) 將安裝所有相依性、安裝 Playwright，然後執行測試。

```yml title=".github/workflows/playwright.yml"
name: Playwright Tests
on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]
jobs:
  test:
    timeout-minutes: 60
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - name: Setup dotnet
      uses: actions/setup-dotnet@v4
      with:
        dotnet-version: 8.0.x
    - name: Build & Install
      run: dotnet build
    - name: Ensure browsers are installed
      run: pwsh bin/Debug/net8.0/playwright.ps1 install --with-deps
    - name: Run your tests
      run: dotnet test
```

#### 透過容器

GitHub Actions 支援使用 [`jobs.<job_id>.container`](https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions#jobsjob_idcontainer) 選項[在容器中執行工作](https://docs.github.com/en/actions/using-jobs/running-jobs-in-a-container)。這對於不汙染主機環境的相依性以及在不同作業系統上為例如螢幕截圖/視覺回歸測試提供一致的環境非常有用。

```yml title=".github/workflows/playwright.yml"
name: Playwright Tests
on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]
jobs:
  playwright:
    name: 'Playwright Tests'
    runs-on: ubuntu-latest
    container:
      image: mcr.microsoft.com/playwright/dotnet:v1.54.0-noble
      options: --user 1001
    steps:
      - uses: actions/checkout@v4
      - name: Setup dotnet
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: 8.0.x
      - run: dotnet build
      - name: Run your tests
        run: dotnet test
```

#### 在部署時

這將在 [GitHub Deployment](https://developer.github.com/v3/repos/deployments/) 進入 `success` 狀態後開始測試。像 Vercel 這樣的服務使用這種模式，因此您可以在他們部署的環境上執行端對端測試。

```yml title=".github/workflows/playwright.yml"
name: Playwright Tests
on:
  deployment_status:
jobs:
  test:
    timeout-minutes: 60
    runs-on: ubuntu-latest
    if: github.event.deployment_status.state == 'success'
    steps:
    - uses: actions/checkout@v4
    - name: Setup dotnet
      uses: actions/setup-dotnet@v4
      with:
        dotnet-version: 8.0.x
    - run: dotnet build
    - name: Ensure browsers are installed
      run: pwsh bin/Debug/net8.0/playwright.ps1 install --with-deps
    - name: Run tests
      run: dotnet test
      env:
        # This might depend on your test-runner
        PLAYWRIGHT_TEST_BASE_URL: ${{ github.event.deployment_status.target_url }}
```

### Docker

我們有一個[預建的 Docker 映像](./docker.mdx) ，可以直接使用或作為更新現有 Docker 定義的參考。 請確保遵循[建議的 Docker 設定](./docker.mdx#recommended-docker-configuration) 以確保最佳效能。

### Azure Pipelines

對於 Windows 或 macOS 代理，不需要額外的設定，只需安裝 Playwright 並執行您的測試。

對於 Linux 代理，您可以使用[我們的 Docker 容器](./docker.mdx)搭配 Azure Pipelines 支援[執行容器化工作](https://docs.microsoft.com/en-us/azure/devops/pipelines/process/container-phases?view=azure-devops)。或者，您可以使用[命令列工具](./browsers#install-system-dependencies)來安裝所有必要的相依性。

要執行 Playwright 測試，請使用此管線任務：

```yml
trigger:
- main

pool:
  vmImage: ubuntu-latest

steps:
- task: UseDotNet@2
  inputs:
    packageType: sdk
    version: '8.0.x'
  displayName: 'Use .NET SDK'
- script: dotnet build --configuration Release
  displayName: 'Build'
- script: pwsh bin/Release/net8.0/playwright.ps1 install --with-deps
  displayName: 'Install Playwright browsers'
- script: dotnet test --configuration Release
  displayName: 'Run tests'
```

#### Azure Pipelines（容器化）

```yml
trigger:
- main

pool:
  vmImage: ubuntu-latest
container: mcr.microsoft.com/playwright/dotnet:v1.54.0-noble

steps:
- task: UseDotNet@2
  inputs:
    packageType: sdk
    version: '8.0.x'
  displayName: 'Use .NET SDK'

- script: dotnet build --configuration Release
  displayName: 'Build'
- script: dotnet test --configuration Release
  displayName: 'Run tests'
```

### CircleCI

在 CircleCI 上執行 Playwright 與在 GitHub Actions 上執行非常相似。 為了指定預建的 Playwright [Docker image](./docker.mdx), 只需在您的設定中使用 `docker:` 修改代理定義，如下所示：

```yml
executors:
  pw-noble-development:
    docker:
      - image: mcr.microsoft.com/playwright/dotnet:v1.54.0-noble
```

注意：使用 docker 代理定義時，您將 playwright 執行的資源類別指定為 'medium' 級別[這裡](https://circleci.com/docs/configuration-reference?#docker-execution-environment)。Playwright 的預設行為是將 workers 數量設定為偵測到的核心數（在 medium 級別的情況下為 2）。將 workers 數量覆寫為大於此數量將導致不必要的逾時和失敗。

### Jenkins

Jenkins 支援管線的 Docker 代理。 使用 [Playwright Docker 映像](./docker.mdx) 在 Jenkins 上執行測試。

```groovy
pipeline {
   agent { docker { image 'mcr.microsoft.com/playwright/dotnet:v1.54.0-noble' } }
   stages {
      stage('e2e-tests') {
         steps {
            sh 'dotnet build'
            sh 'dotnet test'
         }
      }
   }
}
```

### Bitbucket Pipelines

Bitbucket Pipelines 可以使用公開的 [Docker 映像作為建置環境](https://confluence.atlassian.com/bitbucket/use-docker-images-as-build-environments-792298897.html). 要在 Bitbucket 上執行 Playwright 測試，請使用我們的公開 Docker 映像 ([see Dockerfile](./docker.mdx)).

```yml
image: mcr.microsoft.com/playwright/dotnet:v1.54.0-noble
```

### GitLab CI

要在 GitLab 上執行 Playwright 測試，請使用我們的公開 Docker 映像 ([see Dockerfile](./docker.mdx)).

## 快取瀏覽器

不建議快取瀏覽器執行檔, ，因為還原快取所需的時間與下載執行檔所需的時間相當。 特別是在 Linux 下， [operating system dependencies](./browsers.mdx#install-system-dependencies) 需要安裝，這些相依性無法快取。

如果您仍想在 CI 執行之間快取瀏覽器執行檔，請在您的 CI 設定中針對 Playwright 版本的雜湊值快取[這些目錄](./browsers.mdx#managing-browser-binaries)。

## 偵錯瀏覽器啟動

Playwright 支援 `DEBUG` 環境變數以在執行期間輸出偵錯日誌。 將其設定為 `pw:browser` 有助於偵錯 `Error: Failed to launch browser` errors.

```bash
DEBUG=pw:browser dotnet test
```

## 執行有頭模式

預設情況下，Playwright 以無頭模式啟動瀏覽器。 請參閱我們的[執行測試](./running-tests.mdx#run-tests-in-headed-mode) 指南以了解如何在有頭模式下執行測試。

在 Linux 代理上，有頭執行需要安裝 [Xvfb](https://en.wikipedia.org/wiki/Xvfb) 。 我們的 [Docker 映像](./docker.mdx) 和 GitHub Action 都預先安裝了 Xvfb。 要使用 Xvfb 在有頭模式下執行瀏覽器，請在實際命令前新增 `xvfb-run`。

```bash
xvfb-run dotnet test
```
